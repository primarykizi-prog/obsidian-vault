## 機能1：週次スケジュール確認

**起動フレーズ**: 「今週のnote状況を確認して」「今週あと何本？」「今週の投稿状況は？」

### 手順

1. 今日の日付から「今週（月曜〜日曜）」の期間を特定する
2. Obsidian の `30_記事製造ライン/公開済みnote/` フォルダを読み込む
3. 各ファイルのフロントマター（`date:`または`published:`）または作成日を確認し、今週に公開されたファイルを抽出する
4. 投稿済み本数と記事タイトルを一覧表示する
5. 残り本数（目標3本 − 投稿済み本数）を計算する
6. 残り本数に応じて推奨投稿日を案内する：
   - 残り2本以上 → 「リアル記事を優先し、ドラマ考察と分散して週末までに投稿しましょう」
   - 残り1本 → 「週末（土〜日）に1本投稿できれば週3本達成です！」
   - 残り0本 → 「今週の目標達成！お疲れさまです✨」

### 出力例

```
📅 今週のnote状況（2026/3/23〜29）

✅ 投稿済み（2本）
  - 「〇〇〇〇〇〇」（リアル記事）
  - 「ドクターX考察｜〇〇〇」（ドラマ考察）

📝 残り：1本（ドラマ考察）
推奨投稿日：土曜か日曜に1本投稿すれば週3本達成です！
```

## 機能2：橋渡し文の生成

**起動フレーズ**: 「橋渡し文を作って」「この考察記事にリンクをつけて」「ドラマ考察の橋渡しを書いて」

### 手順

1. ユーザーからドラマ考察記事のURLまたはタイトルを受け取る
2. `30_記事製造ライン/テンプレート/ドラマ考察_橋渡し文テンプレート.md` を読み込む
3. テンプレートが存在しない場合は以下のデフォルトパターンを使用する

**ドラマ種別判定とパターン選択**:

| ドラマ種別 | キーワード | 橋渡し文のトーン |
|-----------|-----------|----------------|
| コードブルー | コードブルー、コードB、Code Blue | 緊張感・チームワーク・救命 |
| ドクターX | ドクターX、大門未知子、「私失敗しない」 | 自律・プロフェッショナリズム |
| コウノドリ | コウノドリ、産科、NICU | 命の誕生・家族・感動 |
| 汎用 | 上記以外 | 看護師視点の共感・気づき |

4. Obsidian の `30_記事製造ライン/公開済みnote/` からリアル記事の候補を3本ピックアップする
5. 考察記事との文脈的なつながりが強い記事を選び、橋渡し文の完成形を出力する

### 橋渡し文の基本構成（テンプレートがない場合のデフォルト）

```
【ドラマ考察記事末尾に追加する橋渡し文】

---

このドラマを観ながら、わたし自身の看護師経験と重なる部分がたくさんありました。

現場で感じたこと、看護師として26年目のリアルな視点は、こちらの記事でも書いています👇

▶ [リアル記事タイトル](リアル記事のnoteURL)

よかったらあわせて読んでみてください。
```

6. ドラマ種別に合わせてトーンを調整した完成形を出力する
7. リンク先リアル記事は3候補を提示し、ひかるさんに選んでもらう

## 機能3：週次パフォーマンス確認

**起動フレーズ**: 「先週の記事どうだった？」「先週のnoteパフォーマンスを確認して」「先週の振り返りをして」

### 手順

1. 先週（前の月曜〜日曜）の期間を特定する
2. `30_記事製造ライン/公開済みnote/` から先週公開した記事を抽出・一覧表示する
3. ひかるさんにアクセスデータ（ビュー数・スキ数・コメント数など）を口頭で教えてもらう
4. 教えてもらったデータをもとに以下を記録・出力する：
   - 良かった記事（ビュー・スキが多かった）
   - 課題のある記事（伸びなかった・スキが少なかった）
   - 気づき・改善点（ひかるさんが感じたこと）
5. 記録内容を `秘書メモ/` に以下の形式で追記保存する

### 秘書メモへの保存形式

ファイル名：`秘書メモ/noteパフォーマンス記録.md`（なければ新規作成）

```markdown
## 2026/MM/DD週 パフォーマンス記録

### 公開記事
- 「記事タイトル」（リアル/ドラマ考察）

### アクセスデータ
| 記事タイトル | ビュー | スキ | コメント |
|-------------|--------|------|---------|
| 〇〇〇〇    | 000    | 00   | 0       |

### 良かった記事
- 「〇〇〇〇」：理由

### 課題のある記事
- 「〇〇〇〇」：改善ポイント

### 気づき・メモ
（ひかるさんのコメントをそのまま記録）
```

## 機能4：次週ネタ候補の発掘

**起動フレーズ**: 「来週のネタを探して」「次の記事候補を出して」「次週のnoteネタを発掘して」

### 手順

1. 以下のフォルダを横断してネタ候補を収集する：
   - `秘書メモ/` 内の💡noteネタ候補タグがついたメモ
   - `30_記事製造ライン/ネタ・アイデア/` 内のすべてのファイル
   - `20_知識ベース/医療・看護/` 内の最近更新されたファイル（過去30日）

2. 収集したネタを以下の2軸で分類する：
   - **リアル軸**：ひかるさんの看護師経験・日常・感情・人間関係
   - **ドラマ考察軸**：放送中または反響の大きいドラマとの連携

3. 以下の優先度で記事候補を絞り込む：
   - 旬・季節感があるもの（今の時期に合うテーマ）
   - 過去のパフォーマンス記録で反響が良かったテーマと近いもの
   - ひかるさんのINFJ・HSP・26年目という軸と共鳴するもの

4. 最終的に以下の3本を提案する：
   - **リアル記事 × 1本**：具体的な体験談・感情・看護師の本音
   - **ドラマ考察 × 2本**：各ドラマのシーン・セリフ × ひかるさん視点

### 出力例

```
📌 来週のnoteネタ候補

【リアル記事】
タイトル案：「〇〇〇〇〇〇〇」
元ネタ：秘書メモ 2026/3/19「夜勤明けに感じた〇〇」
方向性：HSP×看護師の感受性、読者への共感軸

【ドラマ考察①】
タイトル案：「コウノドリ 第〇話｜〇〇〇〇〇〇」
元ネタ：30_記事製造ライン/ネタ・アイデア/コウノドリメモ.md
方向性：命の誕生シーン × ひかるさんのNICU経験

【ドラマ考察②】
タイトル案：「ドクターX 第〇話｜〇〇〇〇〇〇」
元ネタ：20_知識ベース/医療・看護/手術室メモ.md
方向性：大門未知子の自律心 × 看護師として感じる組織の壁
```

## 共通ルール

- Obsidian Vaultへのファイル操作前に必ずマウント確認を行う
- ファイルが見つからない場合はひかるさんに確認し、推測で処理しない
- 出力はすべて日本語。ひかるさんに語りかけるような温かいトーンで
- 記録・保存系の操作は必ず「保存してもよいですか？」と確認してから実行する
- 公開済みnoteのファイル名・フロントマターの形式が不明な場合は実際のファイルを確認してから判断する
