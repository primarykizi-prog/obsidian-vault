# Writing-Plans スキル

## 概要
企画書や戦略ドキュメントを「明日から動けるタスクリスト」に変換するスキル。
Week1〜4の週別計画と、各タスクのステップ・完了基準を含む実行計画書を生成する。

## 実行手順

### Step 1: 企画書の読み込み
- 指定されたファイル・テキスト・または直前のbrainstorming出力を読み込む
- 目標・ターゲット・戦略の3つを必ず確認する

### Step 2: タスクの洗い出し
以下の観点でタスクを抽出する：
- **環境構築タスク**（プロフィール設定、ツール準備など）
- **コンテンツ作成タスク**（投稿・記事・素材など）
- **交流・拡散タスク**（リプ、フォロー、コラボなど）
- **計測・改善タスク**（KPI確認、振り返りなど）

### Step 3: 週別スケジュールに配置
```
Week1：基盤づくり（環境整備・初期コンテンツ）
Week2：実行開始（ルーティン確立・交流開始）
Week3：拡張（連携・新フォーマット挑戦）
Week4：振り返り・次月計画
```

### Step 4: 各タスクにステップと完了基準を設定
各タスクは以下のフォーマットで記載：
```
**タスク名**
- ステップ1：〇〇
- ステップ2：〇〇
- 完了基準：〇〇の状態になったらOK
```

### Step 5: 日次ルーティンの提案
- 1日の所要時間に合わせた「毎日やること」をまとめる
- 朝・夜などの時間帯別に整理する

### Step 6: 承認と保存
- 実行計画書を提示して「この計画でよいですか？」と確認
- 承認後、Obsidianの適切なフォルダに保存する

## 出力フォーマット
```markdown
# [プロジェクト名] 実行計画書

## Week1〜4 タスク一覧
...

## 毎日のルーティン
...

## KPI管理表
...
```

## 注意事項
- タスクは「今日からできる」粒度に分解する
- 1日30分以内に収まるよう調整する
- ひかるさんの既存スキル（x-post-generatorなど）を積極的に組み込む
