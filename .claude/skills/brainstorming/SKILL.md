# Brainstorming スキル

## 概要
ユーザーのアイデア・課題をヒアリングしながら深掘りし、構造化された企画書（設計ドキュメント）を生成するスキル。質問は1つずつ行い、回答に応じて次の質問を変化させる。

## 実行手順

### Phase 1: コンテキスト探索
- Obsidian Vaultの関連ノート（日報・秘書メモ・自分OS・知識ベース）を参照して背景情報を収集する
- ユーザーのプロフィール（`10_自分OS/_自分取扱説明書.md`など）を確認する

### Phase 2: 質問フェーズ（5問）
以下の流れで質問を1問ずつ行う。前の回答内容に応じて質問を調整すること。

**質問テンプレート（状況に応じて変更してよい）：**
1. ジャンル・テーマ・対象プラットフォームは何か
2. ターゲットとなる読者・視聴者・顧客は誰か
3. 現状の状況・課題・制約（時間・リソース等）は何か
4. 成功の定義・KPI・達成したい具体的な数字は何か
5. 競合・差別化ポイント・自分の強みは何か

**ルール：**
- 質問は必ず1問ずつ。まとめて聞かない
- 回答がA/B/C選択式でも自由記述でも受け付ける
- 「D: その他（具体的に）」の選択肢を常に含める
- 回答内容によって次の質問を柔軟に変える

### Phase 3: アプローチ提案・確認
5問終了後、以下を行う：
1. 「こういう方向性で進めますがよいですか？」と2〜3のアプローチ案を提示
2. ユーザーが承認したら Phase 4 へ進む
3. 修正依頼があれば調整して再確認

### Phase 4: 企画書（設計ドキュメント）生成
以下のセクションを含む構造化された企画書を生成する：

```
# 企画書タイトル

## 1. 前提条件・背景
## 2. ポジショニング・コンセプト
## 3. ターゲット定義
## 4. コンテンツ構成・戦略
## 5. 成長戦略・KPI（フェーズ別）
## 6. 実行・アクションプラン（初月）
## 7. 今後の拡張・調整方針
```

生成後、「この企画書でよいですか？writing-plansスキルで実行計画に落とすこともできます」と伝える。

## 注意事項
- ユーザーへの最初の返答は「ブレインストーミングを開始します」と明示する
- 一度に複数の質問をしない
- 企画書は日本語で出力する
- ひかるさんのObsidianの文脈・蓄積知識を積極的に活用する
