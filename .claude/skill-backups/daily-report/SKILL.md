# 日報作成スキル

## 目的

ユーザーが提供するメモ（音声メモ・Google Keepメモ・箇条書きなど）を、
Obsidianの日報フォーマットに整形し、Vaultの日報フォルダに直接保存する。

---

## 保存先

```
{VAULT_PATH}/日報/YYYY-MM-DD.md
```

日付はユーザーが指定した日付、または今日の日付（`date +%Y-%m-%d` で確認）を使う。

---

## 日報フォーマット（厳守）

以下のテンプレートを必ず使うこと。絵文字・セクション名・構造は変えない。

```markdown
🎯 戦略・優先順位（AI評価より）

- [ ] **ランクA（絶対）**: [[関連ノート]]を使ったObsidianリンク付きの行動項目
- [ ] **ランクB（進行）**: [[関連ノート]]を使ったObsidianリンク付きの行動項目
- [ ] **ランクC（余裕）**: [[関連ノート]]を使ったObsidianリンク付きの行動項目

💊 一撃の処方箋（ボトルネック対策）

今日のメモの核心課題を1〜2文で鋭く要約。「〜なので〜すること」という処方箋形式で。
Obsidianリンク [[]] を使って関連概念をリンクする。

⚖️ 戦略マトリクス（今日の重要課題）

|優先度|やること|なぜ大事か|
|---|---|---|
|絶対|[[関連ノート]]アクション|理由・目的|
|本命|[[関連ノート]]アクション|理由・目的|
|余裕|[[関連ノート]]アクション|理由・目的|

🌙 21:00の鉄則

- [ ] [[デジタルデトックス]]（画面を閉じる・ストレッチ）
- [ ] 翌朝6時のルーティン確認

💡 AIフィードバック・資産の種

今日のメモから読み取れる気づき・パターン・成長ポイントを2〜4文で。
ユーザーの強みや次のステップを具体的に言語化する。
Obsidianリンク [[]] で関連概念をつなぐ。
```

---

## メモ解析のコツ

音声メモやGoogle Keepのテキストは口語・誤字・断片的な情報が多い。以下の方針で整理する：

- **感情・不安・迷い** → 💊処方箋と💡フィードバックで受け止めて構造化する
- **やりたいこと・アイデア** → 🎯優先順位とマトリクスに落とし込む
- **繰り返し出てくるキーワード** → `[[]]`でObsidianリンクにする（例：`[[妻の退職]]`、`[[AIエージェント]]`）
- **ランクA/B/Cの判断基準**：A=今日絶対やること、B=進行中・少し進める、C=余裕があれば

---

## 実行手順

1. ユーザーからメモと日付を受け取る（日付がなければ今日の日付を使う）
2. メモの内容を分析し、主要テーマを3〜5個抽出する
3. フォーマットに従って日報を作成する
4. 保存先パスを確認する：
   ```bash
   ls "{VAULT_PATH}/日報/"
   ```
5. ファイルを保存する：
   ```
   {VAULT_PATH}/日報/YYYY-MM-DD.md
   ```
6. 保存完了をユーザーに伝える

---

## 注意点

- ファイルが既に存在する場合は上書き前に確認する
- Obsidianリンク `[[]]` は実際にVaultに存在しそうな概念・人物・テーマに使う
- 一撃の処方箋は「〜なので〜すること」という行動指針の形にする
- AIフィードバックはユーザーの言葉をそのまま繰り返すのではなく、Claudeとしての洞察を加える
