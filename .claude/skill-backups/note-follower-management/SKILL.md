# noteフォロワー交流管理スキル

## 目的

ひかるさんがnoteで出会った人たちとの交流を**脳のリソースを使わずに**深められるようにする。
「誰と・いつ・何をした・次は何をする」をObsidianに蓄積し、
大事な人を忘れずに、自然なタイミングで関係を育てていけるようにサポートする。
さらに、交流が深まった人を**紹介シリーズ記事**として定期的に取り上げ、
リストが溜まりすぎず、かつダブりなく、心地よいペースで発信できるよう管理する。

---

## 保存先

```
{VAULT_PATH}/30_SNS管理/noteフォロワー/
├── _交流リスト.md          ← 全員の一覧と次のアクション（メインダッシュボード）
├── _紹介シリーズ管理.md    ← 紹介シリーズのバックナンバー・候補・タイミング管理
├── _紹介記事テンプレート.md ← 紹介記事を書くときのひな形
└── 人物/
    └── 〇〇さん.md          ← 1人1ファイルの詳細ノート（必要に応じて作成）
```

### パス解決ルール（重要）

ユーザーのObsidian Vaultパスは以下に**固定**されている：
- **Windowsパス**：`C:\Users\user\Documents`
- **Obsidian Vault**：`C:\Users\user\Documents\Obsidian Vault`

**毎回のセッション開始時に必ず実行：**
1. `mcp__cowork__request_cowork_directory` ツールで `C:\Users\user\Documents` をマウントする（すでにマウント済みの場合もエラーにならないので必ず実行する）
2. マウント後のVMパスを確認：`find /sessions/*/mnt/Documents -maxdepth 1 -name "Obsidian Vault" -type d 2>/dev/null`
3. 見つかったパスを `{VAULT_PATH}` として使う
4. フォルダが存在しない場合は `mkdir -p` で作成
5. `30_SNS管理/noteフォロワー/` フォルダがなければ `mkdir -p` で作成

## 対応するアクション

リクエスト内容から自動判断して、以下のいずれかを実行する。

### アクション1：新規登録「この人登録して」

新しい交流相手をObsidianに登録する。

**必要な情報（わからない項目はスキップしてOK）：**
- アカウント名（note上の名前）
- note記事URL（あれば）
- どんな人か（一言メモ）
- きっかけ（コメントもらった・記事読んだ・スキされたなど）

**実行内容：**
1. `_交流リスト.md` の「アクション待ち」表に1行追加
2. 「次のアクション」欄に初期アクションを設定（例：記事を読みに行く・コメントを返す）
3. 必要なら `人物/〇〇さん.md` を作成（交流が深くなりそうな人）

---

### アクション2：交流を記録「〇〇さんからコメントきた」「〇〇さんの記事にコメントした」

交流の記録を更新する。

**実行内容：**
1. `_交流リスト.md` の該当行の「最終交流日」「交流メモ」を更新
2. 「次のアクション」を更新（コメント返した → 相手の記事チェック、など）
3. `人物/〇〇さん.md` がある場合は交流ログに追記

---

### アクション3：リスト確認「フォロワーリスト見せて」「今週絡みに行く人は？」

現在の交流状況を見やすく提示する。

**表示内容：**
- 全員のリストと最終交流日
- 「次のアクション」がある人をピックアップ
- 長期間交流がない人（30日以上）をハイライト

---

### アクション4：アクション管理「〇〇さんに次何する？」「アクション未完の人は？」

次のアクションを確認・更新する。

**実行内容：**
- アクションが設定されている人の一覧を出す
- 完了したアクションをユーザーと確認して更新

---

### アクション5：コメント返し「〇〇さんのコメントに返信して」「返信文作って」

自分のnote記事にもらったコメントへの返信文を、ひかるさんらしい言葉で生成する。
生成後に交流記録も自動で更新する（記録のために別途「保存して」と言わなくてOK）。

**必要な情報：**
- 相手の名前（アカウント名）
- もらったコメントの内容（貼り付けてもらえるとベスト）
- コメントされた記事のテーマ（あれば）

**返信文の作り方：**
- ひかるさんらしい温かみのある口調（です・ます調、でも柔らかく）
- AIっぽい定型文にならないよう、相手のコメントの内容を必ず拾う
- 長すぎない（3〜5文が理想）
- 相手が「読んでよかった」「また来たい」と感じる締め方にする
- 複数案（2〜3パターン）を出して選んでもらう

**例（返信文のイメージ）：**
> 〇〇さん、読んでくださってありがとうございます。
> 「△△」という部分に共感してくれてすごく嬉しかったです。
> 私も同じことを感じてずっとモヤモヤしていたので、言葉にしてよかったなと思いました。
> またいつでも遊びにきてください☺

**実行内容：**
1. 返信文を2〜3パターン生成してチャットに表示する
2. `_交流リスト.md` の該当行を更新（最終交流日・次のアクション）
3. `人物/〇〇さん.md` があれば交流ログに追記（「コメントもらい返信した」）
4. 完了報告：「〇〇さんへの返信案を3パターン出しました。交流記録も更新しました。」

---

### アクション6：マガジン追加登録「〇〇さんがマガジンに追加してくれた」

マガジン追加はスキより一段上の行動（わざわざ自分のマガジンにまとめてくれた＝本当に価値を感じてくれている）なので、**紹介シリーズの最優先候補**として扱う。

**実行内容：**
1. `_交流リスト.md` の「マガジン追加してくれた人 ★最優先★」セクションに追加
2. 「紹介シリーズ対象」セクションにも同時に追加（ステータス：候補★★★）
3. `人物/〇〇さん.md` を作成して「紹介ネタメモ」セクションに記録
4. 完了報告：「〇〇さんをマガジン追加者として登録しました。紹介シリーズの最優先候補に入れました。」

---

### アクション7：紹介シリーズ管理「次に誰を紹介する？」「紹介のタイミングは？」「紹介管理して」「〇〇さんの紹介記事を公開した」

紹介シリーズのダブりチェック・候補提案・投稿タイミングの判定、および紹介済み記録を行う。

**ステップ1：ダブりチェック**
`_紹介シリーズ管理.md` のバックナンバーを読み込み、すでに紹介済みの人を把握する。
「紹介シリーズ対象」の候補リストとバックナンバーを突合して、ダブりがないか確認する。

**ステップ2：次の候補を選ぶ**
以下の優先順位で候補を選定する：
1. **★★★最優先**：`_交流リスト.md` の「マガジン追加してくれた人」セクションにいる人
2. **★★★次点**：`_紹介シリーズ管理.md` の候補リストで優先度★★★の人
3. **★★☆候補**：継続交流中で交流が3回以上ある人
4. **★☆☆候補**：アクション待ちから特に深く共鳴した人

**ステップ3：投稿タイミングを判定する**
`_紹介シリーズ管理.md` のバックナンバーの最新公開日を確認し、以下のルールで判定：
- **前回から2週間以内**：「まだ少し早いです。〇〇日頃が次の投稿タイミングです。」
- **前回から2〜4週間**：「そろそろ投稿タイミングです！次の候補は〇〇さんがおすすめです。」
- **前回から4週間以上 / まだ一度も投稿していない**：「タイミングです！最初（または次）の紹介は〇〇さんがおすすめです。」

**ステップ4：結果をチャットに表示する**

```
📋 紹介シリーズ現状レポート（YYYY-MM-DD）

✅ 紹介済み（バックナンバー）
  - vol.1 〇〇さん（YYYY-MM-DD）
  ※まだ紹介記事はありません

🎯 次の紹介候補（ダブりなし確認済み）
  1位：〇〇さん ★★★（マガジン追加・理由：...）
  2位：△△さん ★★★（理由：...）
  3位：□□さん ★★☆（理由：...）

⏰ 投稿タイミング
  前回投稿：YYYY-MM-DD（〇日前）
  → 判定：今がタイミングです！ / あと〇日待ちましょう

📝 次のアクション
  →「〇〇さんの紹介記事を書く」なら：_紹介記事テンプレート をコピーして下書き開始
```

**ステップ5：紹介済みにする場合**
「〇〇さんの紹介記事を公開した」と言われたら：
1. `_紹介シリーズ管理.md` のバックナンバーに追記（vol番号・公開日・記事URL）
2. `_交流リスト.md` の「紹介シリーズ対象」でステータスを「紹介済み（vol.〇）」に更新
3. `人物/〇〇さん.md` の「紹介ネタメモ」の紹介ステータスを「紹介済み（vol.〇 YYYY-MM-DD）」に更新
4. 完了報告：「vol.〇 〇〇さんの紹介を記録しました。次の候補は△△さんがおすすめです。」

## 実行時の注意

- **確認なしで保存してOK**：「登録して」「記録して」と言われたらすぐに保存する。プレビュー確認は不要。
- **情報が少なくてもOK**：アカウント名だけでも登録できる。足りない情報は後から追加する。
- **ダブり防止を必ず守る**：紹介済みの人を候補に出さない。バックナンバーを必ず確認してから候補提案する。
- **マガジン追加は最優先**：マガジン追加してくれた人は問答無用で★★★候補に入れる。
- **ステータス判断はこちらで**：「アクション待ち」「継続交流中」「紹介シリーズ対象」「紹介済み」のどこに入れるかはClaudeが文脈から判断してOK。ユーザーに確認不要。
- **紹介タイミングは自動判定**：「紹介管理して」と言われたら、バックナンバーの日付を確認して自動でタイミングを判定する。ユーザーに日付を聞かない。
