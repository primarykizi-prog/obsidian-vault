# NotebookLM → Obsidian 学習知識変換スキル

## 目的

こういちさんがNotebookLMで噛み砕いた学習内容を、Obsidianの知識倉庫に蓄積する。
「貼るだけ・確認するだけ」で保存できることが最優先。
**ただし「保存して終わり」ではなく「次のアクションにつながる形」で保存すること。**

---

## 保存先（20_知識ベースに統合済み）

```
{VAULT_PATH}/20_知識ベース/YouTube学習/YYYY-MM-DD_テーマ名.md
```

### パス解決ルール（重要）

ユーザーのObsidian Vaultパスは以下に**固定**されている：
- **Windowsパス**：`C:\Users\user\Documents`
- **Obsidian Vault**：`C:\Users\user\Documents\Obsidian Vault`

**毎回のセッション開始時に必ず実行：**
1. `mcp__cowork__request_cowork_directory` ツールで `C:\Users\user\Documents` をマウントする（すでにマウント済みの場合もエラーにならないので必ず実行する）
2. マウント後のVMパスを確認：`find /sessions/*/mnt/Documents -maxdepth 1 -name "Obsidian Vault" -type d 2>/dev/null`
3. 見つかったパスを `{VAULT_PATH}` として使う
4. フォルダが存在しない場合は `mkdir -p` で作成してから保存

日付は今日の日付（`date +%Y-%m-%d`で確認）を使う。

---

## 実行手順（必ずこの順番で）

### Phase 1：分析と変換
1. NotebookLMのアウトプットテキストを受け取る
2. 内容を分析し、テーマ名を自動判定する

### Phase 2：リンク検証（★保存前に必ず実行）
3. **Vault内の実在ファイルを検索する**（「リンク検証ルール」参照）
4. 検証済みのリンクだけを使ってObsidian用マークダウンに変換する
5. **必ず🎯課題定義セクションを含める**（テンプレート参照）

### Phase 3：確認と保存
6. **プレビューをチャットに表示する**
7. 「Obsidianに入れますか？」と確認する
8. YesならObsidianに保存する

### Phase 4：後処理（★保存後に必ず実行）
9. **課題トラッカーに新規課題を追記する**（「課題トラッカー連携」参照）
10. 保存完了をシンプルに伝える
11. **「どれか今やりますか？」と課題の即時実行を提案する**

## Obsidian用マークダウンのフォーマット（厳守）

```markdown
# 🎓 [テーマ名・タイトル]

**元ソース**: YouTube → 文字起こし学習
**保存日**: YYYY-MM-DD
**タグ**: #知識ベース #YouTube学習 #[自動判定タグ1] #[自動判定タグ2]

---

## 💡 この学習の核心

（一番大事なことを2〜3文で。こういちさんが後で見て「あ、そういうことか」とわかる形で）

---

## 📝 重要ポイント

- （ポイント1）
- （ポイント2）
- （ポイント3）
※ 全部じゃなく「こういちさんの事業・発信・人生に使える部分」だけ抜粋

---

## 🔗 こういちさんへの活用メモ

（この知識がMIGAKI事業・note発信・NurseXブログ・自分OSのどこに使えるか1〜2文で）
※ リンクは必ず「リンク検証ルール」に従って実在ファイルのみ記載すること

---

## 🎯 課題定義（この学習から次にやること）

この動画を学んだだけでは意味がない。以下の問いに答えることが次のステップ。

### 課題①：[具体的な課題タイトル]
- [問いかけ・考えるべきこと]
- → **アクション**：[具体的に何をするか、どのファイルに書くか]

### 課題②：[具体的な課題タイトル]
- [問いかけ・考えるべきこと]
- → **アクション**：[具体的に何をするか]

（課題は2〜4個。多すぎると動けない。少なすぎると学びが浅い。）

---

## 関連リンク

- [[実在ファイル名のみ]]（リンク検証済み）
```

### 課題定義の品質基準

課題の質がこのスキルの生命線。以下を守ること：

- **「note記事を書く」「X投稿する」レベルの汎用アクションは禁止**。そんなのは毎日やってること
- 必ずこういちさんの具体的な状況に接続すること：
  - MIGAKI事業（糸魚川・民泊・旅館業許可・体験設計）
  - 看護師キャリア（25年・救急・管理職を断る選択）
  - 自分OS（マルチタイムフレーム目標・勝利条件・習慣化）
  - 発信（note・X・NurseXブログ）の具体的な記事やテーマ
- アクションには**「どのファイルに書くか」または「何を作るか」**を含めること
- 「検討する」「考える」は弱い。「書き出す」「追記する」「作成する」が正解

## リンク検証ルール（★最重要★）

**保存前に必ず実行すること。想像でリンクを書くことは厳禁。**
これが過去に壊れたリンクが97個も発生した原因。二度と繰り返さない。

### Step 1: Vault内のファイル一覧を取得
```bash
find "{VAULT_PATH}" -name "*.md" -type f 2>/dev/null | sed 's|.*/||; s|\.md$//' | sort
```

### Step 2: 関連リンクを選定
内容のテーマに合うファイルを上記リストから**実在確認済みのものだけ**選ぶ。

よく使うリンク先（全て実在確認済み）：
| ファイル名 | いつリンクするか |
|-----------|----------------|
| `_自分取扱説明書` | 自己理解・価値観・判断基準の話 |
| `マルチタイムフレーム目標管理` | 目標設定・キャリア・時間軸の話 |
| `非日常体験リスト20` | 習慣化・行動変容・チャレンジの話 |
| `MIGAKI_コアコンセプト` | 事業戦略・一棟貸し・高単価の話 |
| `MIGAKIブランドストーリー_UX設計` | 体験設計・航海士・おもてなしの話 |
| `MIGAKI_体験コンテンツ案` | 体験メニュー・ミッション型宿泊の話 |

同じYouTube学習フォルダ内のテーマが近いファイルへのクロスリンクも有効。
ただし**ファイル名は日付プレフィックス込みの正確な名前**を使うこと。
例：`[[2026-04-05_40代からの人生経営戦略]]`（`[[40代からの人生経営戦略]]`はNG）

### Step 3: 絶対やってはいけないこと
- 存在確認せずにリンクを書く
- `[[年収アップマニュアル]]` のような「ありそうだけど存在しない」リンクを書く
- 日付プレフィックスなしでYouTube学習ファイルにリンクする
- 迷ったらリンクしない（壊れたリンク1個 > リンクなし10個）

## 課題トラッカー連携（★重要★）

保存完了後、必ず以下を実行：

1. `{VAULT_PATH}/20_知識ベース/YouTube学習/_課題トラッカー.md` を読む
2. 今回の課題定義から抽出したタスクを、適切なカテゴリの末尾に追記する
3. フォーマット：`- [ ] タスク内容（出典：ファイル名）（完了日：　）`

カテゴリの振り分け：
| テーマ | カテゴリ |
|--------|---------|
| MIGAKI事業関連 | 🏠 MIGAKI事業 |
| note/X発信関連 | 📝 note/X発信 |
| ブログ関連 | 💻 NurseXブログ |
| 習慣/ルーティン/自己理解 | 🧠 自分OS |
| 看護師関連 | 🏥 看護師キャリア |
| その他学習 | 📚 知識・学習 |

## 注意点

- **要点は絞る**：NotebookLMの出力が長くても、全部入れない。こういちさんが後で見て使える量だけ
- **活用メモは必ず書く**：「どこで使えるか」がないと死蔵になる
- **課題定義は必ず書く**：「次に何をするか」がないと学びが行動にならない
- **`#知識ベース` タグは必ず入れる**：横断検索のキーとなるタグ
- **Obsidianリンク `[[]]` は実在ファイルのみ**：「リンク検証ルール」に必ず従う。想像で書くな
- **課題トラッカーへの追記を忘れない**：保存して終わりではない
- フォルダが存在しない場合は作成してから保存する
- ファイルが既に存在する場合は上書き前に確認する
- プレビュー確認なしに保存しない（必ず「入れますか？」を挟む）
