# note → Obsidian 知識変換スキル

## 目的

ひかるさんが「いいな」と思ったnote記事を、自分の知識資産としてObsidianに蓄積する。
「渡すだけ・確認するだけ」で保存でき、**後から確実に引き出せる**ことが最優先。

---

## 保存先

```
{VAULT_PATH}/20_知識ベース/note引用/YYYY-MM-DD_記事タイトル_note.md
```

### パス解決ルール（重要）

ユーザーのObsidian Vaultパスは以下に**固定**されている：
- **Windowsパス**：`C:\Users\user\Documents`
- **Obsidian Vault**：`C:\Users\user\Documents\Obsidian Vault`

**毎回のセッション開始時に必ず実行：**
1. `mcp__cowork__request_cowork_directory` ツールで `C:\Users\user\Documents` をマウントする（すでにマウント済みの場合もエラーにならないので必ず実行する）
2. マウント後のVMパスを確認：`find /sessions/*/mnt/Documents -maxdepth 1 -name "Obsidian Vault" -type d 2>/dev/null`
3. 見つかったパスを `{VAULT_PATH}` として使う
4. フォルダが存在しない場合は `mkdir -p` で作成
4. フォルダが存在しない場合は `mkdir -p` で作成してから保存

日付は今日の日付（`date +%Y-%m-%d`で確認）を使う。

---

## 実行手順（必ずこの順番で）

1. 記事を受け取る（URLまたはテキスト）
2. URLの場合 → `web_fetch`で記事本文を取得
3. 記事を分析し、Obsidian用マークダウンに変換する
4. **プレビューをチャットに表示する**
5. 「Obsidianに入れますか？」と確認する
6. Yes なら以下を順番に実行する：
   - **a. 記事ファイルを保存**：`{VAULT_PATH}/20_知識ベース/note引用/YYYY-MM-DD_タイトル_note.md`
   - **b. _INDEX.mdに1行追記**：`{VAULT_PATH}/20_知識ベース/note引用/_INDEX.md` の該当テーマのテーブルに追記
   - **c. 関連MOCに追記**：記事テーマに応じて下記「MOC追記ルール」に従い関連ノートにリンクを追記
7. 保存完了を伝える（「記事ファイル・INDEX・MOCの3か所に保存しました」と明示する）

## Obsidian用マークダウンのフォーマット（厳守）

```markdown
# 📚 [記事タイトル]

**出典**: [著者名]さんのnote
**URL**: [記事URL]
**保存日**: YYYY-MM-DD
**タグ**: #知識ベース #note引用 #[自動判定タグ1] #[自動判定タグ2]

---

## 💡 核心メッセージ

（この記事で一番言いたかったこと・ひかるさんの知識になる部分を2〜3文で）

---

## 📝 要点メモ

- （重要ポイント1）
- （重要ポイント2）
- （重要ポイント3）
※ 全部じゃなく「ひかるさんに刺さる部分」だけ抜粋

---

## 🔗 ひかるさんへの活用メモ

（この知識がひかるさんのnote・ブログ・事業のどこに使えそうか1〜2文で）

---

## 関連リンク

- [[]] （Vaultに存在しそうな関連ノートをリンク）
```

---

## _INDEX.mdへの追記フォーマット

`{VAULT_PATH}/20_知識ベース/note引用/_INDEX.md` の該当テーマのテーブルに1行追加する：

```
| YYYY-MM-DD | [[ファイル名（拡張子なし）]] | 核心メッセージ1文 | #タグ1 #タグ2 |
```

テーマ別テーブルが存在しない場合は新しいセクションを追加する。

---

## MOC追記ルール

記事のテーマに応じて、以下のMOCノートの適切なセクションにリンクを1行追記する：

| テーマ | 追記先MOCファイル |
|--------|-----------------|
| 看護・医療現場・AI時代・キャリア | `{VAULT_PATH}/20_知識ベース/医療・看護/看護師エッセイ_知識ハブ.md` |
| ビジネス・経営・スタートアップ・社会課題 | `{VAULT_PATH}/20_知識ベース/ビジネス・経営/_INDEX.md` |
| note・発信・ブログ・マーケティング | `{VAULT_PATH}/40_NurseX戦略/note収益化戦略（叩き台）.md` |
| メンタル・働き方・自己理解・INFJ | `{VAULT_PATH}/20_知識ベース/医療・看護/看護師エッセイ_知識ハブ.md` |

**追記フォーマット（1行）**：
```
- [[ファイル名]] — 核心メッセージ1文（〇〇さんnote引用）
```

複数テーマにまたがる場合は最も関連が強いMOC1か所のみに追記する。

---

## タグ自動判定ルール

**必須タグ**（全ファイル共通）：`#知識ベース` `#note引用`

**テーマタグ**（内容に応じて選択）：

| テーマ | タグ |
|--------|------|
| 看護・医療 | #看護 #医療現場 |
| キャリア・転職 | #キャリア #転職 |
| ビジネス・副業 | #ビジネス #副業 |
| note・ブログ・発信 | #発信 #note |
| メンタル・働き方 | #働き方 #メンタル |
| お金・収入 | #お金 #収入アップ |
| マーケティング | #マーケティング #セールス |
| AI・DX | #AI時代 #医療DX |
| 社会課題・スタートアップ | #社会課題 #スタートアップ |

---

## 注意点

- **要点は絞る**：全文まとめはしない。ひかるさんが後で見て「ああこれね」とすぐ分かるレベルに
- **活用メモは具体的に**：「使えそう」ではなく「どの記事・どの場面で使えるか」を書く
- **INDEX・MOCの追記を忘れない**：ファイル保存だけで終わると後から引き出せなくなる
