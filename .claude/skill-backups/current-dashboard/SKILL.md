# 現在地ダッシュボード スキル

こういちさんが「今どこにいるか」をパッと把握できる一枚資料をWord形式で作る。
情報はObsidian Vaultから取得し、内容が変わったら更新版を再生成する。

---

## ステップ1：Vaultから最新情報を収集する

以下のファイルを優先的に読み、プロジェクトの現状を把握する。

**必ず読むファイル：**
- `Documents/Obsidian Vault/10_自分OS/_claudian-state.md`
  → 現在のフェーズ・最終更新日・進行中プロジェクト一覧

**MIGAKIプロジェクトの場合：**
- `Documents/Obsidian Vault/20_知識ベース/Web新着digest/` 以下の最新digest.md
  → 補助金情報・締め切り・アクションメモ
- `Documents/Obsidian Vault/` 以下のMIGAKI関連ファイル（あれば）
  → 糸魚川物件、判断シートなど

**秘書メモ・日報も参照：**
- `Documents/Obsidian Vault/秘書メモ/` の最新ファイル
- `Documents/Obsidian Vault/日報/` の最新日報

Gmail下書きの状況は直近の会話文脈から読み取る（APIアクセス不要）。

---

## ステップ2：ダッシュボードの構成を決める

読んだ情報から以下のセクションを組み立てる。

1. **📍 タイトル** — プロジェクト名・更新日付
2. **🗺️ 今どこにいるか（フェーズバー）** — 完了済み/現在地/これから をビジュアル表示
3. **⏰ 締め切りカレンダー** — 期日・残り日数・内容（近い順）
4. **✅ 次にやること** — 優先順に番号付きリスト（理由も添える）
5. **📧 コミュニケーション状況** — メール・問い合わせ・返信待ちの一覧
6. **📊 サマリー** — 数字で今日の動きを一目で

セクションが不要な場合（情報がない場合）は省略してよい。
情報が少ない場合でも「現在地が見える」最低限の内容で出力する。

---

## ステップ3：Node.jsスクリプトでdocxを生成する

`scripts/dashboard_template.js` をベーステンプレートとして使う。
ただし、**内容はVaultから読んだ実際のデータで必ず上書きすること**。
ハードコードの例示内容をそのまま使ってはいけない。

### docxライブラリの場所
```
/sessions/lucid-brave-wright/lib/node_modules/docx
```
requireは以下のパスで行う：
```javascript
const { ... } = require('/sessions/lucid-brave-wright/lib/node_modules/docx');
```

### 出力先
```
/sessions/lucid-brave-wright/<プロジェクト名>_現在地ダッシュボード.docx
```
生成後は `/sessions/lucid-brave-wright/mnt/Documents/` にコピーしてユーザーにリンクを渡す。

---

## ステップ4：ユーザーへの報告

ファイルリンクを渡したあと、1〜2文でポイントを添える。
「○○が最優先です」「○日後が締め切りです」など、今すぐ見るべき情報を一言で伝える。

---

## デザインガイド

見やすさを最優先にした配色・構成を維持する。

| 用途 | カラー |
|------|--------|
| 現在地・アクティブ | #4f46e5（インディゴ） |
| 完了済み | #c8f7c5（薄緑） |
| 緊急・最優先 | #fff3f3 背景 + #e74c3c 文字 |
| 注意・中優先 | #fffbf0 背景 + #f39c12 文字 |
| 計画中 | #f0f4ff 背景 + #4f46e5 文字 |
| 未着手 | #e8e8f0 背景 + #aaaaaa 文字 |

フォントは `Hiragino Sans`、本文サイズは20pt（10pt相当）。
ページ余白は上下左右1080 DXA（約1.9cm）。

---

## 更新のタイミング

以下のタイミングで「ダッシュボード更新しますか？」と提案してよい：
- 補助金や問い合わせの状況が変わったとき
- 新しいアクションが決まったとき
- セッション開始時に「前回から何が変わったか」を整理するとき

---

## 注意事項

- Obsidianの情報が古い場合は「Vaultの更新日：○○」を明記する
- 締め切り残り日数は今日の日付から計算する（環境変数 `TODAY` または `date` コマンドで確認）
- メール状況は最新の会話内容を優先する（Vaultより会話が新しい場合）
