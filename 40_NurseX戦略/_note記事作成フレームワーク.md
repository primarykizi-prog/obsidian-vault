# note記事作成フレームワーク（AI時代版）

> 参照：YouTube「2026年に読まれるnote記事の書き方」
> 作成日：2026-03-16

---

## ブログとnoteの役割分担

| | ブログ（NurseX） | note |
|---|---|---|
| 目的 | SEO・アフィリエイト | 読者との信頼関係 |
| 文体 | 分析・構造的 | 体験・感情ベース |
| CTA | 退職代行・転職 | ブログへの誘導 |
| 重要な要素 | H2/H3構造・吹き出し | 失敗・葛藤・感情の揺れ |
| AI要約 | されてもOK（SEOで勝つ） | 要約されても価値が残る記事に |

---

## AI時代に読まれるnote記事の条件

**「AIが要約しても価値が消えない記事」を書く**

消えない価値の例：
- 成功の裏側にあった失敗・思い込み
- その人にしか言えない視点・反論
- 感情の揺れ・葛藤のプロセス
- 体で証明した体験（数字・場面・言葉）

消えてしまう記事の特徴：
- 結論だけ書いてある
- 一般論・どこかで読んだような内容
- ドラマ解説だけで著者の体験がない

## 3ステップ執筆プロセス

### STEP 1：体験の整理（棚卸し）

Claudeに以下を話しかける：

```
テーマ：〇〇
やったこと：
うまくいったこと：
失敗・迷ったこと：
気づいたこと：
自分だけが知っていること：（わからなければ「特になし」でOK）
```

→「特になし」の場合、ClaudeがQ1〜Q3で質問して引き出してくれる

### STEP 2：構成設計（見出しだけ作る）

ルール：
- 冒頭は結論を言わず**場面描写**から入る
- 最重要セクションは他の**1.5〜2倍の文字数**
- 締めは「この著者にしか言えない結論」
- 最終セクションは「今日からできる3つのこと」

### STEP 3：本文執筆

ルール：
- 事実と感情（意見）を分けて書く
- 煽り・誇大表現は使わない
- AIに品質チェックさせてから出力

## note記事の構成テンプレート

```
① 場面描写フック（結論は言わない）
② ドラマ・作品との接続
③ 著者自身の体験（← ここが核）
   - 具体的な失敗・言葉・日付
   - その時の感情
④ 体験から得た気づき・転換点
⑤ 「この著者にしか言えない結論」
⑥ 今日からできる3つのこと
⑦ ブログ元記事へのリンク
```

## スレッド管理のルール

| スレッド | 用途 |
|---------|------|
| ブログスレッド | WordPress HTML・アフィリリライト |
| noteスレッド | note記事の体験整理・執筆 |
| NotebookLMスレッド | スライド・漫画画像生成 |

→ 一つのスレッドに混在させない（コンテキスト圧迫を防ぐ）

## noteスレッド開始時にClaudeに渡すファイル

1. `_自分取扱説明書.md` ← 著者OS・文体ルール
2. `_コアテーマ軸.md` ← 6つのテーマ軸
3. `_note記事作成フレームワーク.md` ← このファイル

## note記事のチェックリスト（投稿前）

- [ ] 著者自身の体験が1つ以上入っているか
- [ ] 失敗・葛藤・感情の揺れが描かれているか
- [ ] 場面描写で始まっているか（結論スタートではないか）
- [ ] 「今日からできること」が最後にあるか
- [ ] ブログ元記事へのリンクがあるか
- [ ] 煽り・誇大表現がないか

## 体験を引き出すための質問テンプレート

```
Q1：〇〇のとき、頭の中で何を考えていましたか？
Q2：うまくいった瞬間、リアルタイムで気づいていましたか？
Q3：他の人のアドバイスで「それは違う」と思ったことはありますか？
```
