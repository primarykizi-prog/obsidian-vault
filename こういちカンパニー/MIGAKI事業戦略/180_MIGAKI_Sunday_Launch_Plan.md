## 🧲 Mission 1｜フィルタリング機能の定義

### なぜ「有料（¥500）」にするのか——ポジティブな「安心の証」として語る

```
❌ 使ってはいけない説明：
「冷やかしを排除するために有料にします」
「本気じゃない人は来ないでください」
→ 拒絶の言語。ネガティブ。

✅ 使うべき説明：
「月500円が、ここを安全な場所にします。
 あなたの安心も守るための、小さな証です。」
→ 招待の言語。ポジティブ。保護の感覚。
```

### フィルタリングが守るもの（2層構造）

**① CEOの安全を守る**
| 守るもの | 説明 |
|--------|------|
| 住所・物件の場所 | 一人で改装中。家族がいる。まだ公開段階ではない |
| 施工中の写真・動画 | 悪意ある使用を防ぐ（転用・無断掲載等）|
| こういちの連絡先 | 月500円の会員だけに安心して開示できる |

**② コミュニティの質を守る**
| 守るもの | 説明 |
|--------|------|
| 会員同士の空気感 | 「本気の人だけ」という場の質が、長期滞在者の満足度に直結する |
| 住人候補の安心 | 「同じ空間にいる人たちが、全員本気で選ばれた人」という信頼 |
| こういちの時間 | 本気でない問い合わせに使う時間がなくなる |

### 言語化テンプレート（記事・SNS・メンバーシップ説明文で使う）

```
「月500円という金額は、
 あなたが本気であることの証であり、
 私があなたを信頼できる根拠です。
 
 ここは選ばれた人だけの場所ではありません。
 選ぶことを決めた人だけの場所です。」
```

## 🔄 Mission 1b｜共創（Co-Creation）の具体的フロー

> **「完成品を貸す」のではなく「何が必要かを一緒に考える」**

### 月次共創サイクル（具体的な運用）

```
【毎月の流れ】

 毎月10日：「今月の一問」投稿（会員限定note記事）
  └ 例：「Starlinkと光ファイバー、どちらが重要ですか？」
  └ 例：「朝、何があれば最高の一日が始まりますか？」
  └ 例：「除雪を一緒にやりたいですか？やりたくないですか？」
  └ フォーム：Googleフォーム（1問・選択式＋自由記述）
         ↓
 毎月20日：回答を集計・分析・CEO自身が読む
         ↓
 毎月末：「あなたの声でこう変えました」報告記事（会員限定）
  └ 例：「67%が除雪体験を『やりたい』と答えました。
         なので冬の朝の除雪を任意参加の儀式にします。」
  └ 例：「朝のコーヒーステーションへの投票が多かったので設置します。」
         ↓
 翌月：変更された施設で新しい体験が生まれる
         ↓
 住人が「自分の意見で場所が変わった」とnoteに書く → 次の会員が来る
```

### 共創が「機能する」ための3条件

| 条件 | 具体的な実装 |
|------|------------|
| **1問だけ聞く** | 複数質問は回答率が激減する。1ヶ月に1問だけ。これを守る |
| **必ず結果を公開する** | 「聞いたまま無視」はコミュニティを壊す。翌月末に必ず公開 |
| **小さくても実行する** | コーヒーメーカー1台でいい。「意見が採用された」体験が全て |

## 📅 Mission 2｜今週の発表スケジュール（4ステップ）

---

### 📌 Step 0｜今日（木曜日）——既存マガジン経由の「事前告知」

**使うチャネル：** noteの既存マガジン「フォローありがとうございます」

**アクション：**
```
1. 「フォローありがとうございます」マガジン読者へ
   「今週日曜日に、MIGAKI BASEの新しい発表があります。
    少し先に、こちらで伝えさせてください。」
   という短い告知記事（200字以内）を投稿する。

2. X（旧Twitter）に同内容を投稿。
   「日曜日、MIGAKI BASEのメンバーシップを開きます。
    本気でリモートワークの環境を変えたい人へ。
    詳しくは金〜日のnoteで。」
```

**狙い：** 既存読者を「温めて」おくことで、日曜日の投稿の初速が変わる。

---

### 📰 Step 1｜金曜日——「安全への執念」記事

**タイトル（確定稿）：**
> 「消防に2回、保健所に4回落とされた。それでも私が作り続ける理由。」

**核心シーン（記事内に必ず入れる）：**
```
・「この壁の厚さが2cm足りない」と言われた瞬間の話
・26年間、2mmの誤差で人の命が変わる現場にいた自分が
  今は2cmの壁の厚さで書き直している——という対比
・それでも続けるのは「安全基準を守ることが、信頼になる」から
```

**末尾CTA：**
> 「明日、MIGAKI BASEの全貌を話します。父の話も含めて。」

**公開設定：** 無料（フック記事）

---

### 🏔 Step 2｜土曜日——「全貌開示」記事

**タイトル（確定稿）：**
> 「父は甲種一等航海士だった。私は救急看護師だ。二人で『生きる場所』を作った。」

**核心シーン（記事内に必ず入れる）：**
```
・父の「不動心」の石碑——航海士が竹刀を握った庭の話
・実家の元民宿が空になった日、何を感じたか

・「豪華な設備じゃない。静かさが、ここの価値だ。」
・【MIGAKI道場・航海士プラン】の発表
```

**末尾CTA：**

**公開設定：** 無料（全貌開示 + メンバーシップ発表）

## 💴 Mission 3｜個人事業主としての実務設計

### 兼業形態の定義（最初の1〜2年）

```
【主軸】看護師として働く（月収安定）
  └ 月●●万円の固定収入を確保
  └ 「生活費は看護師が稼ぐ」という安心がスモールスタートを可能にする

【副軸】MIGAKI BASE を個人事業として運営
  └ 住人3名 × 月12万円 = 月36万円
  └ 「利益は施設の磨き込みと次の投資に回す」
  └ 失敗しても看護師収入で生活できる = リスクを取れる

この二重構造が、最初の判断を正しくする。
「宿だけで生活する」という焦りが最悪の判断を生む。
「看護師がいる」という安心が最良の判断を生む。
```

### 「会員に何が必要かを問う」アンケート設計

> **目的：施設を磨くために何が本当に必要かを、お金を払う前に会員に聞く。**

**第1回アンケート（メンバーシップ開設直後に送る）：**

```
今月の一問：

「MIGAKI BASEで仕事をするとしたら、
 絶対になければ困るものを1つ教えてください。」

選択肢：
A. 高速インターネット（Starlink）
B. 人間工学的な椅子（ハーマンミラー等）
C. 一人になれる完全個室
D. 朝のコーヒーと静けさ
E. その他（自由記述）
```

**第2回アンケート（1ヶ月後）：**

```
「施設に投資する予算が10万円あります。
 次に買うべきものを1つ選んでください。」

選択肢：
A. スタンディングデスク
B. 外部モニター × 2台
C. 防音個室改装
D. 全自動コーヒーメーカー
E. 除雪機（冬季対応）
```

**第3回アンケート（2ヶ月後・長期滞在者向け）：**

```
「180日間ここに住むとしたら、
 自分が持ってくるもの・手放してくるものを教えてください。」

（自由記述）
```

> このアンケートの回答が「次の記事のネタ」になり、
> 「次の設備投資の根拠」になり、
> 「住人候補を引き付けるコンテンツ」になる。
> 1問が3つの仕事をする。
