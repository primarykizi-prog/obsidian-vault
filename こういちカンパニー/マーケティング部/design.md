## 1. 星野リゾートのサイトから学んだデザイン原則

### 1-1. レイアウト構造
- **フルスクリーンヒーロー**：ファーストビューは100vh（画面全体）の写真で圧倒する
- **縦スクロール型**：各セクションを1画面ずつ見せる「シネマティック」な構成
- **余白の多用**：要素間の余白が大きく、高級感と「呼吸感」を演出
- **グリッドレイアウト**：写真＋テキストの2カラム構成（左右交互）
- **スクロール連動アニメーション**：要素がスクロールに合わせてフェードイン

### 1-2. タイポグラフィ
- **和文**：明朝体ベース（Noto Serif JP相当）。見出しは太め、本文は細め
- **英文**：セリフ体（Cormorant Garamond等）。ブランド名やキャッチは英語
- **文字サイズ**：見出しは大きく（3rem〜）、本文は読みやすいサイズ（1rem〜1.1rem）
- **レタースペーシング**：英語見出しは広め（0.2em〜）で高級感

### 1-3. カラーパレット
- 星野リゾートは白背景＋ダーク文字が基本
- MIGAKIでは以下にカスタマイズ：

| 名称 | 色コード | 用途 |
|------|---------|------|
| 墨黒 | `#1a1a1a` | メインテキスト・ナビ |
| 鉄紺 | `#1b2838` | ヒーローオーバーレイ・アクセント |
| 砂色 | `#c4a882` | 見出しアクセント・ボタン |
| 生成り白 | `#f5f0e8` | 背景（和紙風） |
| 純白 | `#ffffff` | セクション交互背景 |

### 1-4. 写真の使い方
- **ヒーロー**：最も美しい風景写真をフルスクリーンで
- **セクション背景**：パララックス（視差効果）で奥行き
- **ギャラリー**：グリッド配置で物件の魅力を網羅的に見せる
- **加工しない**：フィルター最小限。本物感を大切に

### 1-5. インタラクション
- **スクロールフェードイン**：Intersection Observer APIで要素の出現を検知
- **パララックス**：背景画像のスクロール速度を変える（`background-attachment: fixed`）
- **ハンバーガーメニュー**：モバイルファースト。ナビはオーバーレイ
- **予約CTAの常時表示**：固定ヘッダーに予約ボタン

## 3. 技術構成

### 3-1. アーキテクチャ
- **単一HTMLファイル**（CSS・JSインライン）
- 外部依存：Google Fonts（Noto Serif JP / Cormorant Garamond）
- 写真：`photos/` サブフォルダに配置、相対パス参照
- レスポンシブ：モバイルファースト設計

### 3-2. CSS設計
```
- CSS変数でカラー・フォント・スペーシングを一元管理
- セクションごとに min-height: 100vh
- Flexbox / CSS Grid でレイアウト
- @media クエリでブレークポイント管理（768px / 1024px）
```

### 3-3. JavaScript
```
- Intersection Observer → スクロールフェードインアニメーション
- ハンバーガーメニュー開閉
- スムーススクロール（CSS scroll-behavior: smooth）
- ヒーローのケンバーンズ効果（CSS animation）
```

### 3-4. パフォーマンス考慮
- 画像は `loading="lazy"` で遅延読み込み
- フォントは `font-display: swap` で表示ブロック回避
- アニメーションは `will-change` と `transform` でGPUアクセラレーション
