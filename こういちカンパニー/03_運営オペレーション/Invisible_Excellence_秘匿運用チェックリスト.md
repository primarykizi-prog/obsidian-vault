# 🔒 Invisible Excellence｜「見えない卓越」秘匿運用チェックリスト

> **原則：「結果は魔法のように見えるが、実は裏で緻密なAIシステムが回っている」**
> ゲストに届けるのは「感動」だけ。仕組みは永遠に裏側に置く。

---

## ステップ1｜AI生成感の徹底排除

### 🖊 CEOによる「10%上書き」ルール

Claudeが生成した文章は、必ず以下の要素で10%上書きしてから外部に出す。

- [ ] CEOの口癖・言い回しを1か所以上混ぜる（例：「そう、それが全部だ」「磨くってそういうことなんです」）
- [ ] 糸魚川の天候・季節感を1か所以上挿入する（例：「今朝の日本海は凪いでいました」「雪が溶けて庭の石碑が顔を出した日」）
- [ ] 執筆時のCEOの感情を1行加える（喜び・迷い・静けさ…いずれかを選ぶ）
- [ ] AIらしい「対称的な表現」「箇条書きの羅列」を崩して、語りかける文体に変換する

**固定プロンプト（AI指示時に必ず添える）：**
```
このテキストを仕上げてください。ただし条件があります。
「AIが書いた」と一切感じさせない、体温のある日本語にしてください。
読んだ人が「この人が自分の言葉で書いている」と自然に信じる文体で。
```

## ステップ2｜Obsidianの「聖域化」

### 📵 SNS・note・外部への映り込み防止

- [ ] スクリーンショットを撮る前に、Obsidianのウィンドウを完全に閉じる
- [ ] ナレッジグラフ（グラフビュー）は絶対にSNSに出さない
- [ ] タスク管理・日報フォルダの内容はnoteに引用しない（「今日こんなことがあった」と自分の言葉に変換して出す）
- [ ] Claudeとの会話ログ・プロンプト内容は非公開。「AIと対話した」という事実だけを必要に応じて匂わせる
- [ ] こういちカンパニーの会議議事録・各部署のCLAUDE.mdは内部文書のみ

**出してよいもの（魔法の「気配」）：**
```
✅ 「緻密に管理している」という雰囲気
✅ 「なぜかいつも準備ができている」という印象
✅ 「AIをうまく使っている」というざっくりした事実
❌ 具体的なツール名・プロンプト・ファイル構成・システム詳細
```

---

## ステップ3｜アナログの魔法

### ✍️ デジタル→アナログ変換ルール

Claudeが設計し、Obsidianで管理した情報を、ゲストには必ず人間の温度に変換して届ける。

- [ ] チェックイン前メッセージ：Claudeが生成 → CEOが10%上書き → **手書き風のカードとして印刷または直筆で添える**
- [ ] ウェルカムレター（不動心版）：Claudeが英文生成 → 必ず「お父さん確認済み」の由来を一行加える → 印刷して庭の石碑の隣に置く
- [ ] ゲストへのお礼メッセージ：Claudeの下書き → CEOまたは妻が声に出して「これ自分が言うとしたら？」と確認してから送信
- [ ] 手書きカードの一言（退出時）：完全にCEOまたは妻の直筆。AIは使わない。

**「アナログの魔法」の哲学：**
> デジタルで設計した完璧さを、人間の手で届ける。
> ゲストが触れるのは常に「人間」であり、AIは永遠に裏方に徹する。

## ステップ4｜Beds24の「透明化」

### 📧 自動返信メールの人間化

- [ ] 自動返信メールの文末から「このメールはシステムにより自動送信されています」の定型句を**完全削除**
- [ ] 件名から「自動」「Auto」「System」の文字を除去
- [ ] メール冒頭に必ず「○○様」と名前呼びかけを入れる
- [ ] 署名はCEOの実名＋「糸魚川より」を添える
- [ ] 予約確認メールの末尾：「当日、庭でお待ちしています。——こういち」を固定フレーズに
- [ ] チェックインリマインダー：天気予報を1行入れる（「明日の糸魚川は晴れ予報です。庭の鯉たちも待っています」）
