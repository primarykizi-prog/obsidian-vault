# 📦 note記事 コンテンツパイプライン
## 「倉庫から出口まで」全記事の状態を一元管理する

> **使い方**：毎週日曜に開いて「次に出す記事」を1本決める。それだけでいい。
> ステータスを更新しながら使う。完成稿が3本以上あれば安心圏。

---

## ステータス凡例

| 記号 | 意味 | やること |
|------|------|---------|
| 🌱 | ネタのみ（タイトルと方向性だけある） | 秘書メモか日報から素材を探す |
| 📦 | 素材あり（断片が秘書メモ・YouTube等に存在） | 素材を集めてGeminiかClaudeに投げる |
| ✏️ | 下書きあり（Gemini or 執筆中フォルダに原稿） | 仕上げ・文体チェック・画像プロンプト |
| ✅ | 完成稿（そのまま出せる状態） | noteに貼ってヘッダー画像入れるだけ |
| 📤 | 公開済み | アーカイブ管理のみ |


## 📌 組み立てルール（ここが核心）

記事1本を出すために必要なのは、この3つだけ：

```
① どのファイルに素材があるか → このパイプラインで確認
② どのコアテーマ軸で出すか  → MASTER_PLANのテーマ表を見る
③ Claudeに何を頼むか        → 「このファイルを仕上げて」でOK
```

**秘書メモから記事を作るとき：**
→ 「秘書メモの〇月〇日のやつを、軸④の記事にして」とClaudeに投げるだけ

**YouTube学習から記事を作るとき：**
→ `note-real-article`スキルを使う

**Gemini下書きを仕上げるとき：**
→ `note-rewrite`スキルを使う
