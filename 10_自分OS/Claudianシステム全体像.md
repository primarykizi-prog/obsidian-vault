# Claudianシステム全体像

> こういちのAI環境がどうなっているか・何が効率化されたかの全体像メモ。

---

## 🔄 以前と今の違い

### 以前（Before）

```
こういち
  │
  ▼
Claude.ai（毎回白紙からスタート）
  │
  ├─ 「こういちって誰？」→ プロフィールを読む（重い）
  ├─ 「文体は？」→ 文体メモを読む（重い）
  ├─ 「今何してた？」→ 過去ノートを検索（重い）
  └─ Obsidianを都度探索 → コンテキスト大量消費
```

**問題：**
- 毎セッション5〜8ファイルを読み直す
- 同じ情報を何度も説明する
- Obsidianとの接続で毎回許可が必要
- 記事を書くのに「構成→執筆→変換」をすべて手動で指示

---

### 今（After）

```
こういち
  │
  ▼
Claude.ai（GitHubと直接接続）
  │                    │
  │              GitHubのVaultを
  │              直接参照（軽い）
  ▼
CLAUDE.md（自動読込）
  └─ こういちのプロフィール ✅
  └─ 6軸コアテーマ ✅
  └─ 現在地・未処理タスク ✅
  └─ 起動コマンド一覧 ✅
  │
  ▼
サブエージェント（専門家チーム）
  ├─ digest-agent → Web情報収集
  ├─ reflector-agent → 知識をノートに反映
  ├─ planner-agent → 記事構成案を生成
  ├─ writer-agent → 本文を執筆
  ├─ note-publisher-agent → note.com用に変換
  ├─ weekly-agent → 週次知識統合
  └─ session-agent → 引き継ぎ記録
```

**改善点：**
- セッション開始時のコンテキスト読込：ゼロ（自動）
- Obsidianへのアクセス許可：不要
- GitHubから必要なファイルだけ取得：軽い
- 記事製造が1コマンドずつで完結

## 📱 日常の使い方

### Claude.ai（デスクトップ・メイン）

普段の会話・思考整理・相談はここ。
GitHubと繋がっているのでVaultの知識を参照できる。

```
こういちが話す → Claude.aiが応答
（必要なときだけGitHubのVaultを参照）
```

### Claudian（Obsidian内・専門作業）

コマンド1つで専門エージェントが動く。

| コマンド | 何が起きるか |
|---------|------------|
| `digest` | Web新着情報を6軸で収集→Vaultに保存 |
| `a` | digestのアクションメモを各ノートに反映 |
| `p [テーマ]` | アイデアから記事構成案を自動生成 |
| `b` | 構成案から記事本文を自動執筆 |
| `n` | 記事をnote.com投稿用に変換 |
| `weekly` | 今週の知識のつながりをレポート |
| `s` | セッション終了・引き継ぎ記録を更新 |

## 🏭 記事製造ライン（完成形）

```
アイデア・話題
    │
    ▼「p」
planner-agent
→ Vaultを参照して構成案を自動生成
→ 30_記事製造ライン/執筆中/ に保存
    │
    ▼「b」
writer-agent
→ 構成案＋digest知識で本文を執筆
→ こういちの文体で1,500〜2,000字
    │
    ▼（実体験を記入）
こういちが1か所だけ書き足す
    │
    ▼「n」
note-publisher-agent
→ note.com用フォーマットに変換
→ ハッシュタグ生成
→ 投稿直前の状態に整える
    │
    ▼
こういちがnote.comに貼り付けて投稿
```

## 🧠 パーソナライズの積み重ね

毎回の会話・digestでこういちの知識が蓄積される。

```
2026-04-01 digest
  → COO年収1,015万円のデータ
  → INFJの境界線システム化3具体策
  → 訪問看護財務モデル
       ↓ reflector-agentが各ノートに反映
       ↓ planner-agentがネタ候補として認識
       ↓ writer-agentが記事に活用
```

**これが「パーソナライズ」の正体。**
Claudeがこういちの知識・文体・価値観を学習し続けている状態。
