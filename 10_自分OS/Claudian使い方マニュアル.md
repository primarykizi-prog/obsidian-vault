# Claudian使い方マニュアル

> 「何をどこでやれば、一番早いか」がわかる1枚。
> 迷ったらこのファイルを開く。

---

## 3つのAI、使い分けの原則

| 使う場面 | どれを使う | 理由 |
|---------|-----------|------|
| **実体験を思い出す・深掘りする** | Gemini（音声） | 声で話すから記憶が出てくる。Claudianは体験を「作る」のでNG |
| **記事を書く・Vaultを操作する** | Claudian（Claude Code） | サブエージェント10体が全部Vaultに接続。記事製造ラインの中心 |
| **ファイル作成・スキル実行・投稿準備** | Cowork（この画面） | docx・pptx・画像プロンプト・note投稿準備・スケジュール管理 |

**鉄則：体験はGemini、製造はClaudian、仕上げと管理はCowork。**

---

## Claudianの使い方（コマンド一覧）

Claudianはターミナル（Claude Code）で動く。以下のコマンドを入力するだけ。

### 毎日使うもの

| コマンド     | 何が起きるか               | いつ使う                 |
| -------- | -------------------- | -------------------- |
| `digest` | Web検索→Vaultにdigest保存 | 自動（毎朝8時）。手動でも叩ける     |
| `秘書`     | 思いつき・アイデアを秘書メモに保存    | 何か思いついたとき。何でもOK      |
| `x`      | X投稿案を3つ生成            | 自動（毎朝7時・夜7時）。手動でも叩ける |

### 記事を書くとき

| コマンド | 何が起きるか | いつ使う |
|---------|-------------|---------|
| `p` | 記事の構成案を作る | ネタはあるけど構成がないとき |
| `b` | 構成案→本文を書き上げる | 構成案が完成した後。Geminiの体験メモがあれば渡す |
| `n` | 完成した記事をnote投稿用に変換 | 本文が完成した後 |
| `e` | 次に出す記事を戦略的に決める | 自動（月水金9時）。「次は何を書く？」と迷ったとき |

### たまに使うもの

| コマンド | 何が起きるか | いつ使う |
|---------|-------------|---------|
| `a` | アクションメモ→ノートに反映 | 未処理タスクが溜まったとき |
| `s` | セッション引き継ぎ更新 | Claudianセッション終了時 |
| `weekly` | 週次知識統合レポート | 自動（日曜7時） |
| `m` | 眠っている記憶を発掘 | 週1回。ネタ切れのとき |
| `民泊` | 糸魚川プロジェクト管理 | MIGAKI関連の相談 |

## Coworkの使い方（スキル一覧）

Coworkはこの画面。話しかけるだけで動く。

### よく使うスキル

| 言うこと | 何が起きるか |
|---------|-------------|
| 「日報作って」 | 今日の作業を日報フォーマットでObsidianに保存 |
| 「note記事を書き直して」 | 下書き→リライト＋ヘッダー画像プロンプト |
| 「フルパッケージ作って」 | キーワード1つからブログ＋note記事＋画像プロンプト一括生成 |
| 「コメント作って」 | note記事へのコメント文を生成 |
| 「ダッシュボード作って」 | Vault読み込み→現在地をWord一枚にまとめる |
| 「秘書に伝えて」 | 思いつきをObsidianに分類保存 |
| 「セッション終了」 | 今日の成果をObsidianに記録 |
| 「会議開催」 | こういちカンパニーの役員会議 |

## 1日の流れ（理想形）

```
朝 ─────────────────────────────────
7:00  [自動] X投稿案 × 3本生成
7:07  [自動] スキル健全性チェック
8:00  [自動] digest（Web情報収集）→ Vault保存
8:00  [自動] Vaultブリーフィング（秘書が朝の報告）
9:00  [自動] editorial判断（月水金のみ）
9:00  [自動] YouTube学習テキスト・秘書メモの日次処理

昼・夜 ────────────────────────────
あなた → Gemini音声で体験を語る → テキストをClaudianに渡す
あなた → Claudianで「b」→ 記事完成
あなた → Coworkで「リライトして」→ 仕上げ＋画像プロンプト

夜 ─────────────────────────────────
19:00 [自動] X投稿案 × 3本生成（夜の部）
22:00 [自動] 未完了タスク確認→日報に記録
あなた → 「日報作って」でCoworkに日報を依頼
```

## 情報の流れ（回路図）

```
[外の世界]
    │
    ▼
① digest（毎朝）→ Vault 20_知識ベース/Web新着digest/
    │
    ▼
② editorial（月水金）→ Vault 30_記事製造ライン/ネタ・アイデア/editorial_判断ログ
    │                   ※ digest + Vault内ネタ → 「次に出す記事」を決める
    ▼
③ 記事執筆（Gemini体験 → Claudian p → b → n）
    │
    ▼
④ 投稿（note + X）
    │
    ▼
⑤ パフォーマンス記録（秘書に「○スキだった」→ _claudian-state.md）
    │
    ▼
⑥ 次のeditorial判断の材料に戻る → ①に循環
```

**この回路が止まるパターン：**
- digestが無効になっている → ①が止まる → 外の情報が入らない
- editorialが動いていない → ②が止まる → 「なぜこの記事か」の根拠がない
- パフォーマンスを記録しない → ⑤が止まる → 何が刺さったかわからない

## 困ったときの対処

| 状況 | やること |
|------|---------|
| 何を書くか迷う | Claudianで「e」→ editorialが判断 |
| 体験が思い出せない | Geminiに音声で話す |
| ネタが枯れた | Claudianで「m」→ 記憶発掘 |
| ファイルがどこにあるかわからない | Coworkで「ダッシュボード作って」 |
| 全体の現在地を見たい | Coworkで「ダッシュボード作って」 |
| 今日の成果を記録したい | Coworkで「セッション終了」or「日報作って」 |
| こういちカンパニーの方針を決めたい | Coworkで「会議開催」 |

---

## 重要な注意

- **Claudianが体験を作ることは絶対にない。** 体験はGeminiで引き出してから持ち込む
- **「分析なしに動く」は禁止。** editorialの判断を経てから記事を書く
- **パフォーマンスを必ず記録する。** 「○スキだった」と秘書に伝えるだけでいい
- **Coworkで決めたことは、その日中にVaultに保存する。** 保存しないと次回のClaudianが知らないまま動く
