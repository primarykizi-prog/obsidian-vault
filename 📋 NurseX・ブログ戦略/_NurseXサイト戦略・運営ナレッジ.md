## 🔄 旧世代記事のリライト戦略

### リライトが必要な理由
旧世代記事（白黒・アフィリなし）は収益化されていない。
カラー画像・アフィリ追加・v2骨格への更新が必要。

### SEOリスクの認識
- リライト直後はGoogleが再クロール・再インデックスするため、**一時的に順位が下がる可能性がある（2〜4週間）**
- ただし、コンテンツ品質が上がれば中長期で順位は回復・向上する
- 大幅改変よりも「骨格に沿った肉付け＋画像差し替え＋CTA追加」の方がリスクが低い

### リライト優先順位の判断基準（Search Console連動）

| 条件 | 判断 |
|---|---|
| 表示回数が多い・CTRが低い | タイトル・メタ改善が先。本文リライトは後 |
| 表示回数が多い・CTRも高い | 本文強化＋アフィリ追加で収益化を急ぐ |
| 表示回数が少ない | リライトより新規記事の方が効率的 |
| クリック数トップの記事 | 慎重にリライト。301リダイレクトは不要だがURLは変えない |

## 📊 Search Consoleの見方（こういちさん用メモ）
- **CTR高い＋表示回数少ない** → 検索ボリューム自体が小さいキーワード。伸びしろ限定的。
- **CTR低い＋表示回数多い** → タイトルが刺さっていないか、順位が低い。改善余地大。
- **順位7〜10位** → 1ページ目の下位。リライトで5位以内に入ればCTRが一気に上がる。
- 新記事の効果が出るまで：**インデックス後2〜3ヶ月が目安**
