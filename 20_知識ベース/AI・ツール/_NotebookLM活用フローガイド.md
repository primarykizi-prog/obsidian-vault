# NotebookLM × NurseX ブログ活用フロー完全ガイド

> 作成日：2026-03-16
> 目的：ブログHTML完成後に、スライド（漫画イラスト付き）を効率よく生成するための操作手順

---

## 全体の流れ（3パネル×3ステップ）

```
【STEP 1】左パネル「ソース」→ テキストを正しい形式で貼る
         ↓
【STEP 2】中パネル「チャット」→ 構成確認＆記事生成
         ↓
【STEP 3】右パネル「スライド」→ カスタマイズ設定＋漫画テンプレート選択
```

---

## STEP 1：左パネル「ソース」への貼り方

### やること
ブログHTMLから「テキストのみ」を抜き出して貼る。
**HTMLタグは不要。読める日本語テキストだけ貼る。**

### 貼り付けるテキストの構造（これが重要）

処方箋3項目をバラバラにさせないための書き方：

```
■ タイトル
【ドラマ考察】〇〇に学ぶ！〜〜〜

■ 導入（フック）
「もし明日〜」という問いかけ文。

■ 登場人物カルテ
役名・キャスト・特徴など箇条書き

■ 診察室①（テーマ1の分析）
〜の本文テキスト

■ 診察室②（テーマ2の分析）
〜の本文テキスト

■ 本日の処方箋（3つの行動指針）※ここが重要
対策：完璧じゃなくてもチームの要になる3つの行動

①気づく／②動く／③守る

①気づく——「隙を見せる」という技術
行動：完璧を演じるのをやめ、「あ、間違えた！」と笑える場面を意図的に作る。
結果：後輩が「報告しやすい」と感じ、チームのミスが自然と減る。

②動く——「感情の安定」を武器にする
行動：機嫌の波を小さくする。常に「どうした？」と聞ける一定のテンションを保つ。
結果：困った人が自然と集まり、「頼れる存在」としての価値が生まれる。

③守る——「潤滑油」に徹しながら自分も守る
行動：上下・職種間のクッション役を担う。ただし消耗するほど無理はしない。
結果：チームに価値を発揮しながらメンタルも安定する。

■ CTA（行動喚起）
今の環境が辛いあなたへ。上司に一切会わずに退職できる方法があります。

■ カンファレンスまとめ
今回のまとめ文。

■ 締め
コメント欄への誘導文。
```

### ポイント
- `■ 本日の処方箋（3つの行動指針）` という**大見出しで包む**
- ①②③の前に `①気づく／②動く／③守る` という**一行サマリー**を入れる
- これによりNotebookLMが3項目を「1つのグループ」と認識する

---

## STEP 2：中パネル「チャット」での操作

### 手順①：内容確認（必ず最初に行う）

```
このソースの「本日の処方箋」セクションにある
①気づく②動く③守るの3つの行動指針を要約してください。
```

→ 正しく3つが認識されているか確認する。
→ ここで「処方箋が1つだけ」と出てきたら、ソースの貼り直しが必要。

### 手順②：記事の構成確認

```
この記事全体の構成をセクションごとに教えてください。
```

→ スライド生成前に「何がどこにあるか」をAIに把握させる。

### 手順③（必要な場合）：チャットで記事を生成させる

NotebookLMに「この記事を書いて」と頼む場合：

```
このソースをもとに、以下の形式でブログ記事を書いてください。

・対象読者：職場に疲れた一般の人（看護師以外も含む）
・文体：吹き出し会話形式（ひかる先輩・ゆめこ・ナースX）
・処方箋の3項目（①②③）は必ず全部含める
・締めはCTA（退職代行の紹介）で終わる
```

---

## STEP 3：右パネル「スライド資料をカスタマイズ」の操作

### 設定の選び方

| 項目 | 選択肢 | NurseXの推奨 |
|------|--------|------------|
| 形式 | 詳細なスライド／プレゼンターのスライド | **プレゼンターのスライド** |
| 言語 | 日本語 | **日本語** |
| 長さ | 短め／デフォルト | **短め**（処方箋3項目が収まりやすい） |

> **詳細なスライド**は文章が多くなりすぎて3項目が1枚に収まらない。
> **プレゼンターのスライド**のほうがビジュアル重視で3項目を1枚にまとめやすい。

### 「作成するスライドについて説明してください」欄に入力するプロンプト

#### 処方箋3項目を1枚にまとめたい場合：

```
「本日の処方箋」セクションの①気づく②動く③守るの3つを、
1枚のスライドに横3列で並べてください。
見出しは「本日の処方箋：3つの行動指針」として、
各項目に「行動」と「結果」を簡潔に表示してください。
漫画・マンガ風のビジュアルスタイルで作成。
```

#### 4コマ漫画構成にしたい場合：

```
この記事を4枚のスライドで構成してください。
スライド1：問題提起（「天才じゃないと必要とされない？」）
スライド2：ドラマの考察（森本先生の役割）
スライド3：処方箋①気づく②動く③守る（3列レイアウト）
スライド4：CTA（「環境を変える選択肢がある」）
漫画風・白黒・集中線・スクリーントーン表現で。
```

### 漫画テンプレートの使い方（保存済みテンプレート選択画面）

画面下の `BananaNL` ボタンをクリックすると保存済みテンプレートが出る。

- **漫画**：白黒・集中線・擬音あり → NurseXメインビジュアルに使用
- **みやすい**：シンプルなビジネス向け → 処方箋3項目まとめスライドに向いている
- **Manga / Shoujo / S...**：少女漫画系 → 必要に応じて

**推奨：処方箋スライドは「みやすい」で生成してから、アイキャッチは「漫画」で別途作る**

## 処方箋3項目がバラバラになる問題の根本対策

### 原因まとめ
1. ソースで①②③が独立した見出しになっている
2. 「詳細なスライド」モードを使っている（1見出し＝1スライドになりがち）
3. 説明プロンプトに「3項目を1枚に」という指示がない

### 対策チェックリスト
- [ ] ソースで「■ 本日の処方箋（3つ）」という**親見出し**を作った
- [ ] 「プレゼンターのスライド」＋「短め」を選んだ
- [ ] 説明欄に「①②③を1枚に横3列」と明記した

## NurseX全体のNotebookLM活用マップ

```
ブログHTML完成
    ↓
テキスト抽出（HTMLタグ除去）して貼り付け
    ↓
チャットで構成確認
    ↓
スライド生成（処方箋3項目を1枚指定）
    ↓
    ├─ アイキャッチ用：「漫画」テンプレートで1枚生成
    ├─ note用スライド：4枚構成で生成
    └─ SNS用：「短め＋みやすい」で要約スライド
```

## よく使うプロンプト集（コピペ用）

### ソース確認プロンプト
```
処方箋の①②③を確認して、それぞれ「行動」と「結果」を1文ずつ教えてください。
```

### スライド説明欄プロンプト（処方箋3項目まとめ）
```
「本日の処方箋」の①②③を1枚に横3列で表示。
各列に見出し・行動・結果を3行で。
白黒漫画風・集中線・スクリーントーン表現。
```

### スライド説明欄プロンプト（4コマ全体まとめ）
```
記事全体を4枚スライドに：
①問題提起 ②作品考察 ③処方箋3項目（横並び） ④CTA
白黒漫画スタイル・擬音あり・集中線あり。
```
