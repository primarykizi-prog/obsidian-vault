## 💡 この学習の核心

OpenAI創設者Andrej Karpathyが公開した「AutoResearch」は、AIエージェントに自動で「実験→評価→改善→再実験」のループを回させる仕組み。定量評価できる指標があれば、人間が寝ている間にAIが何十回も試行錯誤して改善してくれる。核心は「何を評価指標にするか」の設計力であり、それはドメイン知識に比例する。

---

## 📝 重要ポイント

- **AutoResearchの構造**: 3ファイル構成。①prep.py（データ読み込み・評価設定）②train.py（AIが編集する実験コード）③PROGRAM.md（実験の方向性・制約・評価基準を人間が書く）
- **自己改善ループの流れ**: Claude Code実行 → 実験コード作成 → 実行 → 評価指標で検証 → 改善ならgitで記録 → 次の実験条件を考える → 繰り返し
- **規模感**: 5分間の学習×12回/時間 = 8時間で約100回の自動実験
- **差別化ポイントは「評価指標の設計」**: AIは指標をハックする（本質的でない改善で数値だけ上げる）。何を・どの観点で評価するかの言語化がドメイン知識に依存する
- **応用範囲**: 株価予測に限らず、プロンプト改善、チャットボット改良、スライドデザイン改善など、定量評価できるものなら何でも使える
- **Claude Code限定**: このループを自然言語で構築できるのはClaude Codeならでは

## 🔗 ひかるさんへの活用メモ

この「自己改善ループ」の考え方は、ひかるカンパニーの顧問の動き方そのもの。顧問は「情報を入れて終わり」ではなく「設計を変える→結果を検証→さらに改善」のループを自律的に回す存在。PROGRAM.mdに相当するのが顧問のCLAUDE.md、train.pyに相当するのが各スキルのSKILL.md、評価指標に相当するのがカンパニーのアウトプット（記事の質、ダイジェストの有用度、アクション実行率など）。
