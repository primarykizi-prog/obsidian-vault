# 🎓 Claudeの「Ask User Input」で要件定義を自動化する魔法のプロンプト

**元ソース**: YouTube → NotebookLM学習
**保存日**: 2026-03-28
**タグ**: #知識ベース #YouTube学習 #AI活用 #ツール #プロンプト

---

## 💡 この学習の核心

やりたいことが漠然としている段階でも、Claudeに「不足してる部分があればステップバイステップで1問ずつ確認して。Ask User Inputスタイルでお願い」と伝えるだけで、AIが必要な情報を自分で聞き返してくれる。要件定義をAIに任せることで、プロンプトを書く力がなくても成果物にたどり着ける。

---

## 📝 重要ポイント

- **魔法のプロンプト**: 「こういう指示をしようと思ってるけど、指示で不足してる部分があればステップバイステップでユーザーに1問ずつ確認して。Ask User Inputスタイルでお願い」
- **Claude限定の機能**: Ask User Input（ポップアップ形式の質問UI）はClaudeにしかない。ChatGPTやGeminiにはこの機能がない
- **要件定義の自動化**: 自分の頭の中が漠然としていても、AIが「何を聞けば動けるか」を判断して1問ずつ聞いてくれる。答えるだけで要件が固まっていく
- **実例①：採用応募フォーム**: 「採用募集のための応募フォームを作成したい」＋魔法のプロンプト → 約10分でHTMLフォームが完成
- **実例②：Zoom相談後の自動化フロー**: 文字起こし→議事録スライド作成→メール自動送信まで、1問1答で要件を固めてGASで実装
- **核心メッセージ**: AIが人に寄り添ってきている時代。プロレベルのAI知識がなくても、現場を理解していればAIを使える。ただし動画を見るだけでなく絶対に手を動かすこと

---

## 🔗 ひかるさんへの活用メモ

「Ask User Inputスタイル」は、今ひかるさんがClaudeで使っている各スキル（秘書、note記事作成、フルパッケージ等）のプロンプト設計にも応用できる。たとえばnurse-x-fullpackageで「キーワードだけ入れたら足りない情報を1問ずつ聞いてくれる」フローにすると、記事の品質がさらに上がる。また「現場を知っていればAIは使える」は、2本目の動画（AI活用3要素）の主張と完全に一致しており、note発信の軸「看護師26年×AI」の補強材料になる。
