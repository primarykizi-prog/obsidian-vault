## 📝 インストールの記録

### 手順（実際にかかった時間：30秒ではなかった）

1. 「コード」タブでインストールコマンドを実行
   ```
   git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup
   ```
2. `bun`が未インストールだったので自動インストール（PowerShell経由）
3. gstack 34スキルのインストール成功
4. `~/.claude/commands/` が存在しなかったため `/office-hours` が動かない問題が発生
5. Cowork側から `.md` ファイルを `commands/` フォルダにコピーして解決
6. `/office-hours` 起動成功 ✅

### Windowsでのつまずきポイント
- シンボリックリンクがWindowsでは作成されない場合がある
- 解決策：`~/.claude/skills/gstack/` の `.md` ファイルを `~/.claude/commands/` にコピー

## 🎯 /office-hours で作成した設計書

### プロジェクト名
**Content Publishing Pipeline**

### gstackが分析したひかるさんの状況
1. コンテンツ生成はすでに解決済み（nurse-x-fullpackage、x-post-generatorが対応）
2. 本当の問題は「生成後にWordPress・X・noteへ手動でコピペ投稿すること」
3. note.comはAPIなし → ブラウザ操作（Claude in Chrome）が必要
4. WordPressとXはAPIで自動化できる
5. 公開前に確認ステップが欲しい（全自動ではなく）

### 採用したアプローチ
**B) API Pipeline**
- WordPress API + X API で自動投稿
- note.comはClaude in Chromeでブラウザ操作
- 確認してから投稿するフロー
- 2回の厳しいレビューで20個の問題を発見・修正済み

### 設計書の保存場所（コードタブ内）
```
~/.gstack/projects/Obsidian/user-unknown-design-20260401-185318.md
```
