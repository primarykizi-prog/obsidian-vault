## 📝 インストールの記録

### 手順（実際にかかった時間：30秒ではなかった）

1. 「コード」タブでインストールコマンドを実行
   ```
   git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup
   ```
2. `bun`が未インストールだったので自動インストール（PowerShell経由）
3. gstack 34スキルのインストール成功
4. `~/.claude/commands/` が存在しなかったため `/office-hours` が動かない問題が発生
5. Cowork側から `.md` ファイルを `commands/` フォルダにコピーして解決
6. `/office-hours` 起動成功 ✅

### Windowsでのつまずきポイント
- シンボリックリンクがWindowsでは作成されない場合がある
- 解決策：`~/.claude/skills/gstack/` の `.md` ファイルを `~/.claude/commands/` にコピー

## 🎯 /office-hours で明確になったこと

### 本当に必要なのはこれだけ
**X（Twitter）への毎日の投稿を自動化する**

### 整理された状況
- **x-post-generator** がすでにX投稿文を生成している（解決済み）
- **足りないのは「生成した文をXに自動投稿する」最後の一歩だけ**
- WordPressは画像・アイキャッチ・SEOなど手動確認が必要 → 自動化しない
- noteも同様 → 自動化しない
- Xのみ：テキストだけなので自動化に適している（月500投稿まで無料）

### gstackの価値
- /office-hours で「本当に必要なもの」を引き出せた
- 実際にコードを書くとき /review・/qa でチェックしてくれる
- 今後もコード開発の相談相手として使える

## 🗝️ gstackの主要コマンド一覧

| コマンド | 内容 |
|---------|------|
| `/office-hours` | プロダクト戦略の壁打ち・設計書作成 |
| `/plan-eng-review` | アーキテクチャ確定 |
| `/plan-ceo-review` | 事業化視点のレビュー |
| `/review` | コードレビュー＋自動修正 |
| `/qa` | ブラウザテスト全自動 |
| `/ship` | テスト→PR作成まで一括 |
| `/cso` | セキュリティ監査 |
| `/retro` | 振り返りレポート |
| `/4dev` | ChatGPTにコードのセカンドオピニオンを依頼 |
| `/gstack-upgrade` | gstackを最新版に更新 |
