# 📋 デイリーオペレーション

> ひかるが毎日この会社をどう動かすかの手順書。
> CEOがやることは「秘書に話しかける」だけ。

---

## 基本スケジュール

### 平日（月・火・水・金・土）

**朝（6:00〜7:30 / 1.5時間）**

1. Claudeを開く
2. 秘書に話しかける：「おはよう、今日のやること確認して」
   → 秘書がToDo・昨日の残り・今日のdigestを確認してくれる
3. 「digestまとめて」と一言
   → 秘書 → リサーチ部がdigest処理 → 3トピックス選定・保存・報告
4. 「X投稿作って」と一言
   → 秘書 → マーケティング部がX投稿案を作成 → レビュー → 投稿
5. 残り時間でnote下書きを進める
   → 「noteの下書き進めて」→ 秘書 → note編集部

**夜（帰宅後 / 2.5時間）**

1. 「今日の進捗は？」と秘書に聞く
   → 秘書 → PMが進捗報告
2. メインの制作作業
   → 「ブログ記事書いて」→ 秘書 → ブログ事業部
   → 「note記事書いて」→ 秘書 → note編集部
3. 「日報作って」と一言
   → 秘書がdaily-reportスキルで日報作成・Obsidianに保存
4. 「セッション終了」と一言
   → session-end-obsidianスキルで記録保存

---

### 休日（木・日）

**まとまった時間が取れる日**

1. 朝：平日と同じルーチン（digest・X投稿）
2. 午前：戦略的な作業
   → 「今週の振り返りして」→ 秘書 → PM
   → 「組織について相談したい」→ 秘書と作戦会議
   → 新しい部署やスキルの設計
3. 午後：コンテンツ制作（まとまった記事を一気に書く）
4. 夕方：翌週の計画
   → 「来週の優先順位決めて」→ 秘書 → PM

---

### 夜勤明け

- 無理しない。体力と相談
- やるなら軽いものだけ：digest確認・X投稿・秘書への思考吐き出し
- 重い制作作業は翌日に回す

---

## CEOの口癖チートシート

| 言うだけでOK | 秘書が動く先 |
|---|---|
| 「おはよう、今日何やる？」 | 秘書 → ToDo確認 |
| 「digestまとめて」 | 秘書 → リサーチ部 |
| 「X投稿作って」 | 秘書 → マーケティング部 |
| 「ブログ記事書いて」 | 秘書 → ブログ事業部 |
| 「note記事書いて」 | 秘書 → note編集部 |
| 「進捗は？」 | 秘書 → PM |
| 「日報作って」 | 秘書 → daily-report |
| 「セッション終了」 | 秘書 → session-end |
| 「思いついたんだけど」 | 秘書（メモ → 分類 → 保存） |
| 「○○について調べて」 | 秘書 → リサーチ部 |
| 「組織について相談したい」 | 秘書（作戦会議モード） |

---

## 週次レビュー（日曜日推奨）

PMに「今週の振り返りして」と伝えると：
- 各部署の成果まとめ
- 来週の優先事項の提案
- ボトルネックの報告

---

## 注意事項

- ひかるは秘書にだけ話しかける。部署に直接指示しない
- 最終レビュー（記事の確認・投稿の承認）はCEOが必ず行う
- 「やらなきゃ」と思ったことは全部秘書に投げる。頭に残さない
- 会社の記録はすべてObsidianに残る。忘れても大丈夫
