# 🔄 ひかるカンパニー ワークフロー

> 迷ったらここを見る。勤務パターン別の「何をどの順番でやるか」。

---

## 📌 大前則：ひかるがやること

```
① Claudeを開く（CoworkまたはClaude Code）
② 秘書を起動する（下記参照）
③ 秘書に指令を出す
④ 秘書が部署に振り分けて実行する
⑤ 上がってきたものをレビューする
⑥ OK or やり直しを判断する
```

これだけ。作業は全部AI社員がやる。

### 秘書の起動方法

**方法1：秘書スキルを呼ぶ（推奨）**
> 「秘書、おはよう」「秘書、digestまとめて」

秘書スキルが自動でカンパニーのCLAUDE.mdを読み込んで動く。
（secretary.skillをインストール済みの場合）

**方法2：直接CLAUDE.mdを読ませる**
> 「ひかるカンパニー/secretary/CLAUDE.md を読んで、秘書として動いて」

スキルが使えない場合やカスタムで動かしたい場合はこちら。

### 部署に仕事を振るとき（秘書が自動で行う）

秘書がCEOの指令を聞いて、該当部署のCLAUDE.mdを読み込んでから実行する。
ひかるが部署を直接指定する必要はない。秘書が判断する。

```
ひかる：「ブログ記事書いて」
  → 秘書が blog/CLAUDE.md を読み込み → ブログ事業部として記事制作
ひかる：「digestまとめて」
  → 秘書が research/CLAUDE.md を読み込み → リサーチ部としてdigest処理
```

## 勤務パターン早見表

| パターン | 曜日の例 | 使える時間 |
|---|---|---|
| A. 通常勤務日 | 月・水・金・土 | 朝1.5h ＋ 夜2.5h |

## パターンA：通常勤務日（朝1.5h ＋ 夜2.5h）

### 朝（06:00〜07:30）

```
06:00 Claudeを開く
  │
  ▼
「おはよう、今日のやること確認して」
  │ → 秘書がToDo・昨日の残りを確認
  ▼
「digestまとめて」
  │ → 秘書 → リサーチ部
  │ → 3トピックス選定 → 知識ベースに保存
  │ → noteネタ・ブログネタ・X投稿案を振り分け
  ▼
「X投稿作って」
  │ → 秘書 → マーケティング部
  │ → 投稿案が上がってくる → レビュー → 投稿
  ▼
残り時間：noteの下書きを進める
  │ → 「noteの下書き進めて」→ 秘書 → note編集部
  ▼
07:30 出勤
```

### 夜（帰宅後〜2.5時間）

```
帰宅後 Claudeを開く
  │
  ▼
「今日の進捗は？」
  │ → 秘書 → PM → 進捗報告
  ▼
メイン制作（1.5〜2時間）※どちらか1つ選ぶ
  │
  ├→「ブログ記事書いて（キーワード：○○）」
  │   → 秘書 → ブログ事業部 → fullpackageスキルで制作
  │   → 上がってきたらレビュー
  │
  └→「note記事書いて（テーマ：○○）」
      → 秘書 → note編集部 → note-real-articleスキルで制作
      → 上がってきたらレビュー
  │
  ▼
「日報作って」
  │ → 秘書 → daily-reportスキル → Obsidianに保存
  ▼
「セッション終了」
  │ → session-end-obsidianスキル → 記録保存
  ▼
終了
```

## パターンB：夜勤前の日（朝1.5h ＋ 午前フリー）

```
06:00 Claudeを開く
  │
  ▼
朝ルーチン（パターンAの朝と同じ）
  │ digest → X投稿 → note下書き
  ▼
07:30〜午前中：まとまった制作時間
  │
  ├→ ブログ記事1本がっつり制作
  │   「ブログ記事書いて（キーワード：○○）」
  │
  ├→ note記事1本がっつり制作
  │   「note記事書いて（テーマ：○○）」
  │
  └→ 秘書と相談（組織改善・新しい部署の検討など）
      「組織について相談したい」
  │
  ▼
「日報作って」→「セッション終了」
  ▼
夜勤へ出発
```

## パターンC：夜勤明け（体力次第）

```
帰宅後
  │
  ▼
体力チェック ← ここが最重要
  │
  ├→ 余裕あり：軽い作業だけやる
  │   「おはよう、digestだけまとめて」
  │   「思いついたんだけど（メモ吐き出し）」
  │   「X投稿1本だけ作って」
  │   → 制作作業はしない。インプットとメモだけ。
  │
  └→ しんどい：休む
      → 何もしなくていい。明日は休みだから大丈夫。
  │
  ▼
やった場合は「セッション終了」で記録だけ残す
```

## パターンD：休日（終日フリー）

```
06:00 Claudeを開く
  │
  ▼
朝ルーチン（パターンAの朝と同じ）
  │ digest → X投稿 → note下書き
  ▼
午前：振り返り＆戦略タイム
  │
  ├→「今週の振り返りして」
  │   → 秘書 → PM → 週次レビュー
  │
  ├→「組織について相談したい」
  │   → 秘書と作戦会議
  │
  └→ スキルの改善・新しい部署の設計
  │
  ▼
午後：コンテンツ制作（まとまった時間）
  │
  ├→ ブログ記事を複数本制作
  ├→ note記事を制作
  └→ フルパッケージ（ブログ＋note同時生成）
      「フルパッケージ作って（キーワード：○○）」
  │
  ▼
夕方：翌週の準備
  │
  ├→「来週の優先順位決めて」
  │   → 秘書 → PM
  │
  └→「日報作って」→「セッション終了」
  │
  ▼
終了
```

```

```

```
組織について相談したい
○○について調べて
来週の優先順位決めて
今週の振り返りして
```

## 🔧 使い分けガイド：OpusとSonnet

| タスク | 推奨モデル | 理由 |
|---|---|---|
| 組織構造の設計・相談 | **Opus** | 戦略的な判断・長い文脈の理解が必要 |
| 新しい部署の設計 | **Opus** | 初回の設計はOpusで質を担保 |
| CLAUDE.mdの作成・修正 | **Opus** | エージェントの「人格」を決める重要な作業 |
| ワークフローの改善 | **Opus** | 全体を見て判断する必要がある |
| ブログ記事の制作 | **Sonnet** | 手順が決まっている繰り返し作業 |
| note記事の制作 | **Sonnet** | スキルが手順を持っている |
| X投稿の作成 | **Sonnet** | 短いテキスト生成 |
| digest処理 | **Sonnet** | パターン化された処理 |
| 日報作成 | **Sonnet** | テンプレートに沿った作業 |
| 秘書への思考吐き出し | **Sonnet** | 整理・分類は定型作業 |

> まとめ：**考える仕事はOpus、作る仕事はSonnet**
