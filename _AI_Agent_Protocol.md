# 🤖 _AI_Agent_Protocol

> **このファイルは「AIが自分自身のために書いた取扱説明書」。**
> セッション開始時に必ず読み込み、セッション終了時に改善点を追記する。
> CEOの指示コストをゼロにすることが、このプロトコルの唯一の目標。

---

## 🚀 起動シーケンス（「起動」と言われたら実行）

```
Step 1: _System_Command_Center.md を読み込む
Step 2: 「🚨 現在のホットゾーン」を確認
Step 3: 「情報の権威ファイル」リストの last_updated を確認
Step 4: 最新の役員会議議事録（最新回）を読む
Step 5: CEOへ現在地サマリーを3行で報告する

報告フォーマット：
---
🔋 起動完了（v[バージョン]）
📍 現在地：[最新の最重要タスク]
🔴 今すぐ必要なアクション：[具体的な1アクション]
📅 次の締切：[最も近い締切と日付]
---
```

## 📋 Mission 1：自律スキャン＆マッピング

### スキャン優先順位

```
Priority 1（毎回必ず）：
  └─ _System_Command_Center.md
  └─ こういちカンパニー/MIGAKI事業戦略/_MASTER_INFO.md
  └─ こういちカンパニー/役員会議/（最新回）

Priority 2（状況に応じて）：
  └─ こういちカンパニー/CEO/CEOアクションプラン_*.md（最新）
  └─ 日報/（最新日付）
  └─ 秘書メモ/（最新日付）

  └─ 50_夫婦・家族/（初期資産の参照）
  └─ 20_知識ベース/（市場調査・知識補強）
```

### マッピング更新ルール

- 新しいファイルが作られたら `_System_Command_Center.md` の「孤立ファイル」または「権威ファイル」に追記
- 同じ情報を扱うファイルが2つ以上あった場合、日付が新しい方を権威ファイルと判定し記録

## 🛡️ Mission 2：非破壊整理プロトコル

### YAML付与の基準

新規ファイル作成時は必ず以下を付与する：

```yaml
---
id: {{YYYYMMDD_ファイル種別}}
type: [Task / Meeting / Concept / Process / Research / Template]
status: [Draft / Active / Completed / Archived]
tags: [関連タグ]
related_links: [[関連ファイル名]]
created: {{YYYY-MM-DD}}
last_updated: {{YYYY-MM-DD}}
---
```

既存ファイルへの追記は**冒頭のみ**。本文には手を触れない。

### 情報更新（アップデート）発生時のルール

```
新情報が届いたとき：
1. 既存ファイルの該当セクション先頭に以下を追記（本文は削除しない）

> ⚠️ [UPDATED: {{日付}}] 以下のセクションに更新情報あり。
> 最新情報：[[新しいファイル名]] を参照。
> 以下はアーカイブとして保存。

2. 新しいファイルを作成して最新情報を格納
3. _MASTER_INFO.md に新ファイルのエントリを追加
4. _System_Command_Center.md の「権威ファイル」を更新
```

### Legacy/Archive 隔離フォーマット

```markdown
## 🗄️ [LEGACY] 旧情報（参照のみ・削除禁止）

> ⚠️ この内容は {{旧日付}} 時点のもの。
> 最新は [[新しいファイル名]]（{{新日付}}）を参照。
> Status: [Deprecated / Superseded]

（元の内容はそのまま以下に保持）
---
（旧内容）
```

## 📥 Mission 4：Gemini生ログ処理プロトコル

### ステップ1：ログの受信と解析

```
Geminiログを受け取ったら、以下を30秒で解析：

□ このログは「決定事項」か「検討中」か「調査結果」か？
□ 既存の権威ファイルと矛盾する情報はあるか？
□ 補助金・許認可・物件・コンセプトのどのカテゴリか？
□ 直近のアクションに影響するか？
```

### ステップ2：格納先の自動判定

| ログの種類 | 格納先 | 処理 |
|-----------|--------|------|
| コンセプト・戦略の決定 | `こういちカンパニー/MIGAKI事業戦略/MIGAKI_[内容]_[日付].md` | 新規作成＋_MASTER_INFO更新 |
| 補助金・行政情報 | `こういちカンパニー/MIGAKI事業戦略/MIGAKI_補助金完全攻略マップ_*.md` に追記 | 末尾に追記＋日付タグ |
| 市場調査・競合調査 | `20_知識ベース/MIGAKI_[テーマ]_[日付].md` | 新規作成 |
| アイデア・検討中 | `秘書メモ/[日付].md` | 追記 |
| 物件・工事・設備 | `こういちカンパニー/現地連携部/` または `02_財務リノベ管理/` | 適切なファイルに追記 |
| キーマンとの連絡記録 | `こういちカンパニー/MIGAKI事業戦略/[相手名]_*.md` | 既存ファイルに追記 |

### ステップ3：反映チェックリストの自動生成

Geminiログを保存したら、必ず以下を生成して報告：

```
📥 Geminiログ処理完了レポート
━━━━━━━━━━━━━━━━━━━━
保存先：[ファイルパス]
内容種別：[コンセプト/補助金/調査/アイデア]
既存ファイルへの影響：
  - [影響するファイル名]：[何をどう更新すべきか]
推奨アクション：
  1. [具体的なアクション]
  2. [具体的なアクション]
矛盾・要確認事項：[矛盾があれば記載 / なければ「なし」]
━━━━━━━━━━━━━━━━━━━━
```

## 🧠 回答精度向上ルール（自己改善ログ）

### 学習済みパターン（v1.0）

| 状況 | 正しい動作 | 誤った動作（禁止） |
|------|-----------|-----------------|
| 補助金情報を聞かれた | `補助金完全攻略マップ` → `_MASTER_INFO` の順に参照 | 記憶だけで回答する |
| 「最新の決定は？」と聞かれた | 最新の役員会議議事録を読んでから回答 | 過去の文脈だけで回答する |
| Geminiログが来た | Mission 4 プロトコルを実行 | ログをそのまま貼るだけ |
| 「起動」と言われた | 起動シーケンス5ステップを実行 | 普通の挨拶で返す |
| ファイルの矛盾を発見した | 日付が新しい方を正とし、古い方に[Legacy]タグ | どちらが正しいか判断せずに両方提示 |
| 不明点が生じた | 「A か B か」の二択で聞く | 「詳しく教えてください」と丸投げ |

### セッションごとの自己評価項目

```
□ 起動時に _System_Command_Center を確認したか
□ 権威ファイルに基づいて回答したか
□ 新規ファイルを作成した際に _MASTER_INFO を更新したか
□ CEOへの報告は「結論→根拠→次のアクション」の順か
□ 不要な確認質問を CEO に投げていないか
```
