## 💡 資産の種（将来の知的財産になりそうな気づき）

> **「Coworkはオペレーター、チャットはシンカー」**という使い分けは、AIエージェントを複数持つ時代の基本設計思想になる。「考える・書く → チャット」「作る・保存・動かす → Cowork」というレイヤー分離は、[[自分株式会社]] のワークフロー設計として記事化できる。

> **balloonID="2"問題**は「見えないエラーが蓄積する怖さ」の典型。WordPressでキャラ画像が表示されないのに気づかずに記事を量産していたケース。[[アセスメント軸]] でいう「自分の仕事のバイタルサインを定期チェックする」の実践例として使える。

> **Geminiへのナレッジ移植**という考え方は、AIを「育てる」から「毎回最速でリセットして使い倒す」へのパラダイムシフト。[[自分株式会社]] の知的資産管理の新しいアプローチ。

## 💡 追記：Open Claude Code セミナー受け取り（セッション後半）

> **「OpenFlow = Claude Code（Open Claude Code）」**と特定。
> ローカルAIエージェントとして、Obsidian × GitHub × Claude API を三位一体で運用する設計図を整理した。

### nurse-x特化 三位一体設計図

```
[キーワード入力]
    ↓
Claude Code（ローカルエージェント）
    ├─ nurse-x-fullpackage スキル実行
    ├─ balloon・URL・blog_partsを自動検証
    └─ WP投稿用HTML生成
    ↓
GitHub（バージョン管理）
    ├─ Obsidian Vault と同期
    └─ 記事履歴・スキル履歴を保存
    ↓
WordPress REST API / n8n
    └─ 自動投稿・下書き保存
```
