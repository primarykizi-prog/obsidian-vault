# 🔬 NotebookLM活用ガイド
> **使い方の基本**: ObsidianのノートをNotebookLMに読み込ませて、深掘り・分析・アイデア発掘に使う。書くのはCursor、整理するのはObsidian、**分析・発見するのがNotebookLM**。

---

## 📥 何を読み込ませるか（用途別）

| 目的 | 読み込む資料 |
|---|---|
| 自分の思考を整理したい | 過去のGemini・AI会話のエクスポート |
| 記事の深掘りをしたい | 執筆中ノート（1〜3本） |
| 次の記事アイデアを見つけたい | Google Keepアイデア一覧 ＋ 執筆中ノート一覧 |
| 自分らしさを言語化したい | `_自分取扱説明書` ＋ `_文体・口癖メモ` ＋ 既存記事 |
| テーマの全体像を掴みたい | `_コアテーマ軸` ＋ 関連する執筆中ノート |

---

## 💬 プロンプト集

### 【A】自分の思考・価値観を掘り出す

```
このドキュメントは私（看護師25年・INFJ・こういち）とAIの過去の会話記録です。
以下を分析してください：

1. 私が繰り返し気にしているテーマや悩みは何ですか？
2. 私の価値観や判断基準として一貫しているものは何ですか？
3. 私が最も熱量を持って語っている話題はどれですか？
4. 私が無意識に使っている口癖・フレーズを抽出してください。
```

---

### 【B】記事アイデアを深掘りする

```
添付のノートは私が書きたい記事の下書きです。
以下の視点で深掘りしてください：

1. この記事で一番刺さるメッセージは何だと思いますか？
2. 読者が「自分のことだ」と感じる場面はどこですか？
3. まだ書けていない角度や視点はありますか？
4. タイトル案を5つ提案してください（note向け・読者が思わずクリックするもの）。
```

---

### 【C】複数ノートのつながりを見つける

```
添付のノートは私（看護師・ブログ筆者）が書いた複数の記事・メモです。
以下を教えてください：

1. これらの記事に共通して流れているテーマ・軸は何ですか？
2. 記事同士でつながっているアイデアや概念はありますか？
3. シリーズ化できそうな組み合わせはありますか？
4. まだ書けていない「穴」となっているテーマはありますか？
```

---

### 【D】Gemini会話履歴から自分OSを抽出する

```
このドキュメントは私がAIと行った過去の会話記録です。
私の「思考OS」を定義するために以下を抽出してください：

1. 私がよく使う言い回し・フレーズ（口癖）
2. 私が大切にしている価値観トップ5
3. 私が苦手・嫌いだと感じているもの（バイアス）
4. 私の「熱量の源泉」になっているもの
5. 私の文章のリズム・トーンの特徴

※出力はObsidianに貼り付けられるMarkdown形式でお願いします。
```

---

### 【E】記事の読者ペルソナを作る

```
添付の記事・ノートを読んで、この内容が最も刺さる読者像を定義してください：

1. 年齢・職業・状況（具体的に）
2. この記事を読む前に抱えている悩み・不満
3. この記事を読んだ後に得られる気づき・変化
4. この読者がSNSでシェアしたくなる瞬間はどこですか？

※看護師・医療従事者・転職・キャリアに悩む30〜50代を想定しています。
```

---

### 【G】日記から明日の戦略を出力する

```
以下のフォーマットに従って、最新の日記の内容を分析し、
明日の戦略と行動指針を出力してください。

【制約事項】
1. 出力はすべてMarkdown形式とし、Obsidianにそのまま貼り付けられる状態にすること。
2. 重要なキーワード、人物、目標、または将来「資産」になりそうな概念は、
   必ず [[ ]] で囲んでリンク化すること。
3. 私の「INFJとしての特性（のめり込みやすさ、全体俯瞰）」や、
   「現在の目標（AIを活用した全自動化、自分の時間・余白の確保）」の
   文脈を深く読み取ること。

【出力フォーマット】
## 🎯 戦略・優先順位（AI評価より）
- [ ] **ランクA（絶対）**: （明日最優先でやるべきことを [[ ]] リンク付きで）
- [ ] **ランクB（進行）**: （次に進めるべきことを [[ ]] リンク付きで）
- [ ] **ランクC（余裕）**: （時間が余ったらやることを [[ ]] リンク付きで）

## 💊 一撃の処方箋（ボトルネック対策）
> （思考の浪費や作業の停滞ポイントを指摘し、
>  それを解消する1行アドバイスを [[ ]] リンク付きで記載）

## ⚖️ 戦略マトリクス（今日の重要課題）
| 優先度 | やること | なぜ大事か |
| :--- | :--- | :--- |
| 絶対 | （ランクAの内容） | （本質的な理由。重要な単語を [[ ]] でリンク） |
| 本命 | （ランクBの内容） | （本質的な理由。重要な単語を [[ ]] でリンク） |
| 余裕 | （ランクCの内容） | （本質的な理由。重要な単語を [[ ]] でリンク） |

## 🌙 21:00の鉄則
- [ ] [[デジタルデトックス]]（画面を閉じる・ストレッチ）
- [ ] 翌朝6時のルーティン確認

## 💡 AIフィードバック・資産の種
> （今日の日記から抽出した、将来の「知的財産」になりそうな
>  気づきを、[[ ]] リンクを多用して記述）
```

**使い方**：日記全文をNotebookLMに貼り付け → このプロンプトを入力
**出力の使い方**：出力をコピーして `日報/YYYY-MM-DD.md` の末尾に貼る

### 【F】音声概要（ポッドキャスト機能）用の指示

```
この記事を2人の会話形式の音声概要にしてください。
話し手のキャラクター設定：
- 話し手A：25年の看護師経験を持つ実践者（こういち）
- 話し手B：一般の会社員で、キャリアに悩んでいる30代

会話のトーンは親しみやすく、専門用語は必ず噛み砕いて説明すること。
```

## 🔄 NotebookLMとObsidianの連携フロー

```
① ObsidianのノートをMarkdown（.md）またはテキストでコピー
        ↓
② NotebookLMの「新しいノートブック」に貼り付け or アップロード
        ↓
③ 上のプロンプト集から目的に合うものを使って質問
        ↓
④ 得られた気づき・アイデアをObsidianに貼り返す
   （→ 該当ノートの「💡 エピソード素材メモ」に追加）
        ↓
⑤ Cursorで記事を仕上げる
```

---

## 📌 よくある使い方パターン

**月1回の棚卸し用**
→ その月に書いた記事・メモを全部NotebookLMに読み込んで、プロンプトCで「つながり発見」をする

**新しい記事を書く前**
→ 関連する既存ノートをNotebookLMに読み込んで、プロンプトBで「深掘り」してから書き始める

**Gemini会話が溜まったとき**
→ エクスポートしてプロンプトDで「思考OS抽出」して`_自分取扱説明書`を更新する
