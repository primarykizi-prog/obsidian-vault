> NurseX ブログ → スライド漫画生成 の完全手順

## ◆ 画面の役割を先に理解する

```
┌─────────────────┬──────────────────────┬──────────────────┐
│   左パネル        │     中パネル           │    右パネル        │
│   「ソース」       │     「チャット」         │    「Studio」       │
├─────────────────┼──────────────────────┼──────────────────┤
│ 材料を入れる場所  │  AIと話す場所          │  成果物を作る場所  │
│                  │                       │                  │
│ ブログのテキストを│ 「処方箋を確認して」   │ スライド・動画・   │
│ ここに貼る       │ などと指示する         │ マインドマップなど │
└─────────────────┴──────────────────────┴──────────────────┘
         ①まず材料　→　②内容を確認　→　③成果物を作る
```

## ◆ 手順①：左パネルに材料を貼る

### やること
「＋ ソースを追加」→「テキストを貼り付け」をクリック

### 貼るテキストのルール
**HTMLタグは一切不要。読める日本語のテキストだけ。**

### ⚠️ 処方箋3項目がバラバラになる原因はここ

**目標：①だけ1枚・②だけ1枚・③だけ1枚、合計3枚にする**

問題：①②が同じスライドにまとまったり、③だけ単品になったりする

原因：①②③の見出しの「重さ」がNotebookLMから見て揃っていない

---

❌ **ダメな貼り方（①②が一緒になったり崩れる）**
```
①気づく——「隙を見せる」という技術
行動：〜〜〜
結果：〜〜〜

②動く——「感情の安定」を武器にする
行動：〜〜〜
結果：〜〜〜

③守る——「潤滑油」に徹しながら自分も守る
行動：〜〜〜
結果：〜〜〜
```
→ 見出しの形が揃っていないと、AIがグループ分けを間違える

---

✅ **正しい貼り方（これで3枚に確実に分かれる）**
```
■ 処方箋①：気づく——「隙を見せる」という技術
行動：完璧を演じるのをやめ、失敗を笑える場面を作る
結果：後輩が報告しやすくなり、チームのミスが減る

■ 処方箋②：動く——「感情の安定」を武器にする
行動：機嫌の波を小さくし、常に「どうした？」と聞ける状態を保つ
結果：困った人が集まり「頼れる存在」の価値が生まれる

■ 処方箋③：守る——「潤滑油」に徹しながら自分も守る
行動：上下・職種間のクッション役を担う。無理はしない
結果：チームに貢献しながらメンタルも安定する
```
→ **「■ 処方箋①：」「■ 処方箋②：」「■ 処方箋③：」と完全に同じ形の見出しを揃える**のがポイント

## ◆ 手順②：中パネル（チャット）で確認する

スライドを作る前に**必ず**この確認をする：

### チャット欄にコピペする文（確認用）
```
処方箋の①②③を確認してください。
それぞれ「行動」と「結果」を1文ずつ教えてください。
```

### 確認の判断基準
- ✅ ①②③が3つ全部返ってきた → 次のステップへ進んでOK
- ❌ 1つしか返ってこなかった → ソースの貼り直し（手順①に戻る）

## ◆ 手順③：右パネル「スライド」で設定する

「スライド」ボタンをクリックすると設定ダイアログが出る。

### 設定ダイアログの入力方法

```
┌──────────────────────────────────────┐
│ 形式の選択                            │
│  □ 詳細なスライド                     │  ← 選ばない（文章が多すぎる）
│  ✓ プレゼンターのスライド ← これを選ぶ │
├──────────────────────────────────────┤
│ 言語：日本語　　長さ：短め ← これを選ぶ│
├──────────────────────────────────────┤
│ 作成するスライドについて説明してください│
│                                       │
│  ↓ ここに下のプロンプトをコピペする    │
└──────────────────────────────────────┘
```

### 「説明」欄にコピペするプロンプト

#### プロンプト（処方箋を確実に3枚に分けたい）
```
処方箋の①気づく②動く③守るを、それぞれ独立した1枚ずつ、
必ず3枚のスライドに分けてください。
1スライドに1項目だけ入れてください。
白黒漫画スタイルで作成してください。
```

#### プロンプト（記事全体をスライドにしたい）
```
記事全体をスライドにしてください。
処方箋の①②③はそれぞれ独立した1枚ずつ（計3枚）にしてください。
1スライドに1項目のみ。
白黒漫画風・集中線スタイルで。
```

## ◆ 手順④：BananaNLの漫画テンプレートを選ぶ

ダイアログ左下の **「BananaNL」ボタン** をクリックすると保存済みテンプレートが出る。

```
┌────────────────────────────────────┐
│ 保存済み   ビジネス   人気カタログ  │
├────────────────────────────────────┤
│ 漫画           ← ✅ アイキャッチ用  │
│ みやすい       ← ✅ 処方箋まとめ用  │
│ Manga/Shoujo  ← 少女漫画風（任意） │
└────────────────────────────────────┘
```

**使い分けの基準：**
- アイキャッチ（記事トップ画像）→ **「漫画」**
- 処方箋①②③の各スライド → **「漫画」**（1枚ずつ個別に生成）

テンプレートを選んだら **「生成」ボタン** を押す。

## ◆ 全体の流れ（最終まとめ）

```
STEP 1 ─────────────────────────────────────────────
左パネル「＋ソースを追加」
↓
「■ 本日の処方箋：3つの行動指針（①②③）」の親見出しつきテキストを貼る

STEP 2 ─────────────────────────────────────────────
中パネル「チャット」に確認プロンプトを貼る
↓
①②③が3つ返ってきたら次へ

STEP 3 ─────────────────────────────────────────────
右パネル「スライド」をクリック
↓
形式：プレゼンターのスライド
長さ：短め
説明欄：処方箋3項目を1枚横3列に（プロンプトをコピペ）

STEP 4 ─────────────────────────────────────────────
「BananaNL」→「みやすい」（処方箋用）または「漫画」（アイキャッチ用）
↓
「生成」ボタンを押す
↓
完成！
```

## ◆ うまくいかないときのチェックリスト

| チェック項目 | 確認方法 |
|-------------|---------|
| ソースに親見出し「■ 本日の処方箋：3つ」があるか | 左パネルのソースをクリックして確認 |
| チャットで①②③が3つ返ってきたか | 確認プロンプトを使う |
| 形式を「プレゼンターのスライド」にしたか | 設定ダイアログで確認 |
| 長さを「短め」にしたか | 設定ダイアログで確認 |
| 説明欄に「横3列」と書いたか | 説明欄に明記する |
