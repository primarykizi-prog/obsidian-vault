# ツール関係性メモ

## 対象

- ` _コアテーマ軸.md `
- ` _プロンプト集.md `

## 関係性（結論）

- **`_コアテーマ軸.md`**：**何を書くか／どの方向で語るか**を決めるための **コンパス（判断基準）**
- **`_プロンプト集.md`**：**どうAIに頼むか／どう書かせるか**を安定させるための **操作パネル（手順・呪文）**

→ **軸で方向を決める → プロンプトで実行する**、が基本の噛み合わせ。

## 役割の違い

### `_コアテーマ軸.md`（設計・統一）

- **役割**：記事がブレないように「こういちの記事に流れるコア」を定義（軸①〜⑥）
- **使うタイミング**：記事を書く前／構成を直す前
- **効き方**：
  - 「この話は結局どの軸？」が決まると、主張・事例・結論が揃って文章が締まる
  - 被り回避にも使える（例：軸②が増えすぎたら軸④へ角度変更）

### `_プロンプト集.md`（実行・再現性）

- **役割**：Cursor/Claude/Gemini等に貼り、毎回同じ品質で出力させるテンプレ
- **構造**：Aブレスト／B記事執筆／C自己分析／D日記フィードバック／E NotebookLM
- **ポイント**：
  - Bの「こういちの文体で記事を書く（基本指示）」は、参照ノートに `_コアテーマ軸.md` を含むため、**軸と文体の両方を同時に固定できる**

## 使い方（おすすめの流れ）

### 1) まず `_コアテーマ軸.md` で軸を1つ決める

- 迷うなら「読者が“知ることで救われる”のはどの軸か？」で選ぶ

### 2) 次に `_プロンプト集.md` のB（記事執筆系）で書かせる／整える

- **基本**：B「こういちの文体で記事を書く（基本指示）」
- **派生**：B「参考資料から横展開する（派生記事作成）」（軸①〜⑥への接続指示が明記されている）

### 3) 最後に軸チェックで監査する

- 主張が軸からズレていないか
- いつもの軸（例：軸②）に寄りすぎていないか

## 使い分けのコツ（短く）

- **迷ってるのが“内容・方向性”** → `_コアテーマ軸.md`
- **迷ってるのが“頼み方・文章化”** → `_プロンプト集.md`
- **最短ルート**：軸を1つ決める → プロンプトBで書く → 軸チェックで整える
