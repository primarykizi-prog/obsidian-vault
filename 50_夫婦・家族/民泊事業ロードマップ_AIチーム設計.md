## 🎯 最終ゴール

> **「こういちさんが糸魚川にいなくても、民泊事業が安定して回っている状態」**
>
> - 月次レポートを見て判断するだけ

## 🗺️ ゴールから逆算したフェーズ設計

```
[今] ─── Phase0 ─── Phase1 ─── Phase2 ─── Phase3 ─── [ゴール]
         準備・調査    仕組み構築   試験運用     本格稼働
         〜3ヶ月      〜6ヶ月      〜12ヶ月     12ヶ月以降
```

---

## Phase 0｜準備・調査（〜3ヶ月）

**このフェーズのゴール：「動ける状態にする」**

### やること
- [ ] 糸魚川市役所に旧旅館業許可の状態を確認する
- [ ] 離れの現状調査（エアコン・トイレ・消防設備の確認）
- [ ] 親に「民泊のサポート役をやってもらえるか」を改めて確認・合意
- [ ] 空き家活用補助金・緊急加速化地域の補助金を調べる
- [ ] 民泊管理会社 2〜3社に問い合わせ・見積もりを取る

### AIチームにできること（この段階）
- 補助金制度の調査・整理
- 管理会社への問い合わせメール文の作成
- 市役所への質問リスト作成
- 情報収集・整理・Obsidianへの保存

---

## Phase 1｜仕組み構築（〜6ヶ月）

**このフェーズのゴール：「運営の仕組みと役割を決める」**

### やること
- [ ] 旅館業許可（簡易宿所）の再取得 or 民泊新法で届出
- [ ] 管理会社との契約（委託内容・手数料の交渉）
- [ ] 親の役割マニュアルを作る（チェックイン・清掃確認・緊急対応）
- [ ] 予約プラットフォームの選定（Airbnb / STAY JAPAN など）
- [ ] AIチームの運用設計（下記参照）
- [ ] 離れの初期整備（備品・写真撮影・掲載文の作成）

### AIチームにできること（この段階）
- 掲載文・写真キャプション・タイトルの作成
- 価格設定の調査（競合比較）
- 親向けマニュアルの作成
- ゲスト向けのQ&A・ハウスマニュアルの作成

---

## Phase 2｜試験運用（〜12ヶ月）

**このフェーズのゴール：「離れだけで小さく回して検証する」**

### やること
- [ ] 離れだけで受付開始（Airbnb等で掲載）
- [ ] 初期レビューを積み上げる（最初の3ヶ月が肝）
- [ ] 月次で収支・稼働率・課題をレビューする
- [ ] AIチームの運用を改善しながら安定させる
- [ ] 繁忙期（スキーシーズン）の集客を最大化する

### AIチームにできること（この段階）
- 問い合わせ・予約対応の文章サポート
- レビュー返信の文章作成
- 月次収支レポートの整理
- 季節プロモーションの企画・告知文作成

---

## Phase 3｜本格稼働（12ヶ月以降）

**このフェーズのゴール：「本館も含めて安定収益を確立する」**

### やること
- [ ] 本館の活用方針を決める（合宿受入 / ワーケーション / 賃貸）
- [ ] 合宿・スポーツチームへのB2B営業
- [ ] インバウンド対応（英語・中国語のAI翻訳活用）
- [ ] 収益の一部を設備投資に回す（温泉・BBQ・ワーケーション設備）

## 🤖 AIエージェンシーチームの設計

### チーム全体像

```
【こういちさん】オーナー・意思決定者
      ↓（月1回の確認のみ）
┌─────────────────────────────┐
│      AIエージェンシーチーム      │
│                                   │
│  ①予約管理    ②ゲスト対応       │
│  ③財務管理    ④マーケティング   │
│  ⑤現地連携                       │
└─────────────────────────────┘
      ↓（日常の現地作業）
【お父さん・お母さん】現地サポーター
（チェックイン・清掃確認・緊急対応）
```

---

### エージェント①｜予約管理エージェント

**役割：予約を漏れなく管理し、ダブルブッキングを防ぐ**

| 担当業務 | 内容 |
|---|---|
| 予約確認 | Airbnb / STAY JAPANの予約を毎日確認 |
| カレンダー管理 | 予約状況をこういちさんと親に共有 |
| 問い合わせ対応 | 基本的なQ&Aへの返信文の作成 |
| 空き枠の最適化 | 稼働率が低い時期を検知してアラート |

**使用ツール：** Gmail / Googleカレンダー / Airbnbメッセージ

---

### エージェント②｜ゲスト対応エージェント

**役割：チェックインからチェックアウトまで、ゲスト体験を管理する**

| 担当業務 | 内容 |
|---|---|
| 事前案内 | チェックイン3日前に自動で案内メッセージ |
| ハウスルール共有 | 施設のルール・周辺情報をゲストに送信 |
| 緊急時の初期対応 | 問題発生時の最初の返信（エスカレーション先は親・管理会社） |
| チェックアウト後 | レビュー依頼メッセージの送信 |

---

### エージェント③｜財務管理エージェント

**役割：収支を記録し、月次レポートをこういちさんに届ける**

| 担当業務 | 内容 |
|---|---|
| 収入記録 | 各予約の売上・管理会社手数料を記録 |
| 経費記録 | 備品・修繕・清掃費などを記録 |
| 月次レポート | 月末に収支サマリーを作成・共有 |
| 確定申告準備 | 年間収支データを整理して税理士に渡せる形に |

---

### エージェント④｜マーケティングエージェント

**役割：集客を継続的に改善し、稼働率を上げる**

| 担当業務 | 内容 |
|---|---|
| 価格最適化 | シーズン・競合を見て価格を提案 |
| レビュー管理 | 届いたレビューへの返信文を作成 |
| 季節プロモーション | スキーシーズン前の告知・ワーケーション集客など |
| 掲載文の改善 | 稼働率が低いときに掲載文・写真を見直し提案 |

---

### エージェント⑤｜現地連携エージェント

**役割：お父さん・お母さんとこういちさんをつなぐ**

| 担当業務 | 内容 |
|---|---|
| スケジュール共有 | 次の予約・チェックイン日時を親にわかりやすく連絡 |
| 清掃確認 | チェックアウト後の清掃完了を記録・報告 |
| 備品補充アラート | 備品が少なくなってきたらアラートを出す |
| 緊急連絡フロー | 問題が起きたとき「誰に・何を・どう連絡するか」のフロー整備 |
