# 【運営ドキュメント】noteマガジン交流管理 ワークフロー

> **対象者**: ひかる運営チーム
> **目的**: noteマガジン交流管理の全体フローを標準化し、月次ループで継続的に回す
> **最終更新**: 2026-03-29

---

## 概要

ひかるさんのnote記事が、他のnoteユーザーのマガジンに追加されることがある。
これは「静かな応援」であり、フォロワーとの交流を深める重要な接点。

このドキュメントでは以下を標準化する：

1. マガジン追加者の**発見・記録**の手順
2. **月次チェック**の実施方法（スケジュールタスク連動）
3. **紹介シリーズ記事**の作成・投稿フロー
4. 各ツールの役割と使い方

---

## システム全体図

```
Gmail通知（リアルタイム）
        ↓
note.com API スキャン（月1回・全記事対象）
        ↓
Obsidian _交流リスト.md へ登録
        ↓
人物ノート（/人物/XXXさん.md）作成
        ↓
紹介シリーズ候補選定
        ↓
紹介記事（note下書き）作成 → note投稿
        ↓
紹介済みステータス更新
```

---

## ツール一覧と役割

| ツール | 役割 |
|--------|------|
| **Gmail（MCP）** | マガジン追加通知メールの受信・確認 |
| **note.com API** | 全記事の過去マガジン追加状況の網羅的取得 |
| **Claude in Chrome** | note.com APIへのJavaScript実行（3段階スキャン） |
| **Obsidian** | 交流リスト・人物ノート・紹介下書きの一元管理 |
| **スケジュールタスク** | 月次チェックの自動リマインド |

## STEP 1：マガジン追加者の発見

### 1-A：Gmail通知確認（随時）

Gmailで以下のキーワードで検索：

```
subject:マガジンに追加
```

- 各メールから「noteプロフィールURL」を取得
- 表示名・マガジン名を記録

### 1-B：note.com APIスキャン（月1回）

Gmailでは**過去分や件数の多い追加が漏れる**ため、月次でAPIスキャンを行う。

**手順（Claude in Chromeで実行）:**

**① ひかるさんの記事ページを開く**

```
https://note.com/hikaru_kango26/n/nc49d0d9db7fe
（任意の記事でOK）
```

**② Step1：全記事キーを取得（JSコンソール実行）**

```javascript
(async () => {
  const keys = [];
  for (let p = 1; p <= 8; p++) {
    const r = await fetch(`https://note.com/api/v2/creators/hikaru_kango26/contents?kind=note&page=${p}&per=10`);
    const d = await r.json();
    const c = d.data?.contents || [];
    if (c.length === 0) break;
    c.forEach(n => keys.push(n.key));
  }
  window._step1 = keys;
  console.log('記事数:', keys.length);
})();
```

**③ Step2：各記事のマガジンキーを収集**

```javascript
(async () => {
  const magKeys = new Set();
  for (const nk of window._step1) {
    const r = await fetch(`https://note.com/api/v3/notes/${nk}?draft=false`);
    const d = await r.json();
    (d.data?.belonging_magazine_keys || []).forEach(k => magKeys.add(k));
    await new Promise(res => setTimeout(res, 200));
  }
  window._step2 = [...magKeys];
  console.log('マガジン数:', window._step2.length);
})();
```

**④ Step3：マガジン詳細（オーナー名・マガジン名）取得**

```javascript
(async () => {
  const results = [];
  for (const key of window._step2) {
    const r = await fetch(`https://note.com/api/v1/magazines/${key}`);
    const d = await r.json();
    if (d.data) {
      results.push({
        magKey: key,
        magName: d.data.name,
        owner: d.data.user?.nickname,
        ownerUrl: `https://note.com/${d.data.user?.urlname}`,
        count: d.data.note_count
      });
    }
    await new Promise(res => setTimeout(res, 300));
  }
  window._step3 = results;
  console.log(JSON.stringify(results, null, 2));
})();
```

> **注意**: Step2・Step3はそれぞれ前のStepが完了してから実行すること。

## STEP 2：Obsidianへの記録

### 2-A：_交流リストへの登録

ファイル：`30_SNS管理/noteフォロワー/_交流リスト.md`

「マガジン追加してくれた人 ★最優先★」セクションに追記：

| 項目 | 内容 |
|------|------|
| 名前 | 表示名（nickname） |
| noteURL | `https://note.com/{urlname}` |
| マガジン名 | 追加されたマガジン名 |
| 備考 | 件数・★評価 |
| 紹介ステータス | `候補` |

**★評価基準:**

| 評価 | 条件 |
|------|------|
| ★★★ | 複数回追加 OR 1万件超の大型マガジン |
| ★★ | 医療・看護系 OR 5,000件超 |
| ★ | その他・印象的なマガジン名 |
| （なし） | その他 |

### 2-B：人物ノートの作成

ファイル：`30_SNS管理/noteフォロワー/人物/{urlname}さん.md`

```markdown
# {表示名}さん

- **noteURL**: {URL}
- **表示名**: {nickname}
- **きっかけ**: ひかるさんの記事をマガジンに追加してくれた

## 交流ログ

- {日付}：「{マガジン名}」マガジンに追加

## メモ・印象

（note.comでプロフィールを確認して記入）

## 紹介ネタメモ

- **紹介ステータス**: 未紹介
- **紹介したい理由**:
- **この人の記事で特に好きなもの**: （要確認）
- **紹介記事に使えそうなエピソード**:

## 次のアクション

- [ ] noteプロフィールを確認して表示名・発信テーマを把握する
- [ ] 紹介候補として検討する
```

## STEP 3：紹介シリーズ記事の作成

### 紹介タイミングの目安

- **月1〜2回**のペースで投稿
- 候補が5名以上たまったら紹介記事を作成
- 1記事あたり紹介人数：5〜15名程度

### 紹介記事の構成

参考ファイル：`30_SNS管理/noteフォロワー/紹介記事下書き_マガジン追加してくれた方々_Part1.md`

各紹介のパターン：

```markdown
## {表示名}さん ｜ [note]({プロフィールURL})

**「{マガジン名}」に追加してくださいました。**

{その人への個別メッセージ・エピソード・共鳴した点を2〜4文}

---
```

**末尾の「そのほかにも……」枠（人数が多い場合）:**

```markdown
## そのほかにも……

**Aさん**の「〇〇マガジン」、**Bさん**の「△△マガジン」……
それぞれの言葉で、私の記事を残してくださっていること、本当にありがとうございます。
```

### ファイル命名規則

```
紹介記事下書き_マガジン追加してくれた方々_Part{N}.md
紹介記事下書き_{テーマ}_{日付}.md
```

### リンク形式について

現在の形式（Markdown埋め込みリンク）：
```markdown
[note](https://note.com/{urlname})
```

→ note投稿時に**各人の代表記事リンク**に差し替える（プロフィールページより記事リンクが温かみがある）。
差し替えは手動で行う。

## STEP 4：投稿後の更新

### 4-A：_交流リストの紹介ステータス更新

`候補` → `紹介済み（vol.{N}）` に変更

### 4-B：人物ノートの更新

```markdown
## 紹介ネタメモ

- **紹介ステータス**: 紹介済み（{記事タイトル}・{投稿日}）
```

### 4-C：紹介した相手へのお知らせ

紹介記事を公開したら、相手のnoteにコメントやスキでお知らせする。

> 「先日、マガジンに入れてくださっていたことをnote記事でご紹介させていただきました。ありがとうございます」

## 注意事項

- **note.com APIは公式ドキュメントなし**：仕様変更で取得できなくなる可能性あり。月次スキャンで動作確認を兼ねる
- **belonging_magazine_keysはv3エンドポイント**：v2は別用途。混同しないこと
- **マガジン詳細はv1のみJSON**：`/api/v2/magazines/{key}` と `/api/v3/magazines/{key}` はHTMLを返すため使用不可
- **リクエスト間隔**：APIスキャン時は200〜300msのインターバルを必ず入れる（サーバー負荷軽減）
